## Description
`same_leading_anchor` uses a same-line range comparison, so one comment may match both a parent declaration and a nested expression. Other comments have no matching span and disappear. The formatter has no consumption invariant proving that each concrete comment is emitted once.

## Design
Introduce stable comment identities and exact token-boundary attachment. Remove broad line-based span matching. A formatter pass should expose exact leading, trailing, dangling, and EOF queries and record consumption so tests can reject missing or duplicate trivia.

## Acceptance Criteria
- [x] The one-line function reproduction emits its leading comment once.
- [x] Initializer-leading comments remain attached to the initializer.
- [x] Missing and duplicate comment consumption are observable in formatter tests.
- [x] Existing formatter snapshots remain stable unless correcting broken comment placement.
