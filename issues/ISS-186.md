## Description
Replace the shallow linked-program-to-LoisVM boundary with the agreed whole-program executable pipeline, including ordered top-level initialization, per-instance immutable globals, bytecode schema v2, and consistent interpreter and Wasm execution.

## Design
Extract target-independent linking into `lanec/module/link`, add `lanec/executable` for Whole-Program Elaboration, keep reachability collection in execution-image lowering, and extend LoisVM with Instance Globals and an explicit Instance Initializer. Preserve eager linked top-level initialization and the single-shot execution lifecycle.

## Acceptance Criteria
- [ ] Linking, executable elaboration, and LoisVM lowering follow the documented one-way package dependency graph.
- [ ] Valid Lane top-level values lower to LoisVM bytecode instead of `UnsupportedTopLevel`.
- [ ] Interpreter and Wasm execute the same initializer, entry, ARC cleanup, and failure lifecycle.
- [ ] The authoritative example and full MoonBit validation gates pass.
