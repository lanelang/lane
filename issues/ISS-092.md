## Description
Expose enough linked artifact metadata to validate runtime effect conventions from source-level exported identities. Runtime convention registration should bind names such as `Stdlib.Io.Write.println : (String) -> Unit`, then resolve them to linked Buslane operation identities before execution.

## Design
Use source-level module path, effect name, operation name, payload types, and return type as the validation surface. Do not expose Buslane debug names or raw operation numbers as the host-facing API. Do not require official Stdlib fingerprint pinning. Preserve the metadata through compile, interface/object artifacts, link, binary artifacts, and inspect output as needed for debugging.

## Acceptance Criteria
- [x] Linked program artifacts can answer whether `Stdlib.Io.Write.println(String) -> Unit` exists and map it to a Buslane operation id.
- [x] Validation uses source-level exported identity and signature metadata, not Buslane debug names.
- [x] Artifact text/binary roundtrips preserve the runtime convention metadata.
- [x] Tests cover successful mapping and mismatch cases for module path, effect name, operation name, payload type, and return type.
