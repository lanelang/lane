## Description
The domain glossary and ADRs still describe contextual forwarding fields. After removing field offer, docs must state that contextual resolution consumes only explicit named offers and offered parameters.

## Design
Remove the `Contextual Forwarding Field` term and replace ADR text that endorsed field forwarding with a superseding decision. Clarify that `auto` and `offer` are orthogonal: `auto` only affects call-site insertion, while `offer` places a binding into the current contextual offer environment.

## Acceptance Criteria
- [x] Contextual-resolution glossary has no field-offer terminology.
- [x] ADR documentation records the removal and rationale.
- [x] Roadmap/checklist no longer claims field forwarding is an active completed feature.
- [x] Validation commands and close notes record the completed migration.
