## Description
`parse_concrete` passes each lexer token to Moonyacc as `(token, loc, loc)`. Moonyacc expects separate start and end positions and treats equal positions as zero-width symbols. As a result, generated AST spans often begin at an unrelated nonterminal or collapse to the final delimiter, making exact formatter trivia ownership impossible and weakening diagnostics.

## Design
Project each token span into a zero-width start position and a zero-width end position before invoking the generated parser. Keep lexer token spans unchanged. Add parser regressions for full declaration, block, and expression ranges.

## Acceptance Criteria

- [x] Block and expression spans include opening and closing tokens.
- [x] Existing parser diagnostics remain correctly located.
