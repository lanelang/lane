## Description
Implement Lane source formatting that uses syntax-aware pretty-printing breakpoints to keep ordinary formatted code within the default width without hard-wrapping source text.

## Design
Follow ADR-0059. Formatting is a soft-width contract owned by `lanec` syntax pretty-printing, not by CLI post-processing. The first implementation should use shared formatting primitives, cover common list-like syntax, then cover expression and declaration heads. Do not split identifiers, module paths, comments, or string literals, and do not introduce trailing commas or user-visible width configuration.

## Acceptance Criteria
- [x] Width-sensitive formatting primitives exist and are used by syntax pretty-printing.
- [x] List-like syntax formats consistently in inline and broken layouts.
- [x] Declaration heads and expression chains format consistently in inline and broken layouts.
- [x] Formatter tests cover snapshot output, parse-back, and idempotence.
- [x] Final validation includes syntax/format tests, full tests, examples smoke, formatting/interface generation, and whitespace checks.
