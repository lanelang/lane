## Description
Define the ordinary Buslane type representation used by dictionary CPS before rewriting executable expressions.

## Design
Generate one nominal immutable dictionary shape per concrete effect. Parameterize each dictionary by the effect arguments and answer type, and store statically ordered operation callables as constructor payloads. Replace each kind-Effect parameter with one stable kind-Type companion and provide reusable ordinary context-pack shapes for abstract rows with multiple context terms.

## Acceptance Criteria
- [x] Every concrete effect has exactly one generated dictionary type and constructor.
- [x] Dictionary fields follow declared operation order and have answer-typed CPS callable types.
- [x] Every kind-Effect parameter has one stable kind-Type companion.
- [x] Empty, singleton, and multi-term context types are represented without runtime effect tags.
- [x] Type rewriting preserves pure functions and erases nonempty latent effects into ordinary CPS function types.
- [x] Exact tests cover generic effects, hidden operation parameters, and nested first-class effectful function types.
