## Description
Artifact schema validation, fingerprint checks, and conversions between compiled modules and text artifacts are spread across free functions. They are behavior of specific artifact structs and should be easier to discover from those types.

## Design
Move high-value validation and conversion helpers onto `ModuleInterfaceArtifact`, `ModuleObjectArtifact`, `LinkedProgramArtifact`, and `CompiledModule` where that reduces parameter passing. Keep public command behavior unchanged and do not preserve legacy aliases.

## Acceptance Criteria
- [x] Artifact validation/conversion call sites read through owning methods.
- [x] Compile/link/runlinked artifact flows still pass tests.
- [x] Public API changes are intentional and reflected by `moon info`.
