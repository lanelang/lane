## Description
Source handlers have `with` blocks, operation arms, payload patterns, and a
final branch. Buslane handler alternatives only bind evaluated payload values
positionally, so source payload patterns must be checked and eliminated before
Buslane lowering.

## Design
Add checked handler expressions whose handler tables are grouped by concrete
effect, with resolved operation identities, checked payload pattern matrices,
resume binders, residual effect information, and final-branch data. Reuse the
existing pattern decision machinery to compile payload patterns into Buslane
matching before operation alternatives reach Buslane.

## Acceptance Criteria
- [ ] Source handler syntax/checking represents handled effects, operation
  arms, resume binders, and final branches.
- [ ] Handler arm operation payload patterns are checked for usefulness and
  exhaustiveness per operation.
- [ ] Resume binders receive function types whose latent effect is the handler
  residual effect.
- [ ] Checked handler data groups operation arms by concrete effect and keeps
  source spans outside Buslane.
- [ ] Pattern lowering eliminates source payload patterns before Buslane
  `OperationAlternative` values are created.
- [ ] Typechecker/lowering tests cover direct arms, pattern arms, missing
  operation coverage, duplicate handled effects, and resume typing.
- [ ] `moon test modules/lanec` passes.
- [ ] `moon info && moon fmt` has been run after the change.
