## Description
Resolve effect-level alias bodies and effect annotations that refer to effect declarations, Effect-kind parameters, module-qualified effect names, and effect set aliases.

## Design
Keep resolver expansion-free. Resolve names in effect-level contexts into effect declarations, Effect-kind parameters, or alias references that later checking can expand. Qualified names use `Module.Name`, matching top-level type alias and effect access. Do not introduce `::` syntax for effect aliases.

## Acceptance Criteria
- [x] Resolver recognizes local and module-qualified effect declarations in effect alias bodies.
- [x] Resolver recognizes local and imported effect set aliases in `! UI` and `{ UI, Log }`.
- [x] Resolver preserves alias references for typecheck-time expansion.
- [x] Hover/reference collection remains coherent for effects and aliases.
