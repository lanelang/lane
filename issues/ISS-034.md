## Description
Effect equality cannot be structural array equality because unions must ignore
singleton ordering and duplicate singleton effects while preserving row
variables. Buslane needs a canonical effect model before verifier rules can be
implemented safely.

## Design
Implement canonical effect normalization for union flattening, deduplication,
order-insensitive singleton comparison, and optional row-variable preservation.
Add effect well-formedness checking against metadata and kind information.

## Acceptance Criteria
- [ ] Canonical effects ignore singleton order and duplicate singleton effects.
- [ ] Open effect rows preserve residual `Kind::Effect` row variables.
- [ ] Effect well-formedness rejects unknown effect ids, unknown operation ids
  where applicable, and non-effect row variables.
- [ ] Tests cover closed sets, open rows, duplicates, ordering, and malformed
  effect terms.
- [ ] `moon test modules/buslane` passes.
- [ ] `moon info && moon fmt` has been run after the change.
