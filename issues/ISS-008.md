## Description
Add `textDocument/hover` support to `lane lsp` using semantic hover payloads from `lanec`.

## Design
Advertise `hoverProvider` in initialize capabilities after the semantic analysis API is available. Hover content should be derived from checked types, kinds, and declaration summaries rather than from raw source text. Prefer stable Markdown code blocks or plain strings that do not depend on full pretty-printer layout churn.

## Acceptance Criteria
- [x] `initialize` advertises `hoverProvider`.
- [x] `textDocument/hover` returns `null` for unsupported positions instead of an error.
- [x] Hover over value references shows the checked value type.
- [x] Hover over type references shows kind or nominal declaration summary.
- [x] Hover over fields or variants shows their owner-aware type information where available.
- [x] LSP tests cover request and response JSON for representative hover cases.
