## Description
Compile diagnostics use several free functions whose first argument is a diagnostic type, severity, label, or error kind. These functions obscure ownership and made earlier classification logic harder to reason about.

## Design
Move concrete diagnostic behavior onto methods such as `CompileErrorKind::to_diagnostic`, `CompileDiagnostic::body_doc`, `CompileDiagnosticSeverity::to_doc`, and `CompileDiagnosticLabel::to_doc`. Preserve public rendering behavior and avoid compatibility aliases.

## Acceptance Criteria
- [x] Compile diagnostic construction and rendering are expressed as methods on the owning types.
- [x] Public diagnostic pretty output remains stable.
- [x] Diagnostic golden tests pass.
