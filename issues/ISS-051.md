## Description
Checked Lane handlers have source-level operation arms and payload patterns.
Buslane handlers require tables grouped by handled effect, operation
alternatives with positional payload binders, and explicit resume values.

## Design
Finish the handler lowering boundary introduced by ISS-046. Lower each checked
handler into Buslane handler tables grouped by singleton effect, lower
operation alternatives using the pre-lowered positional payload binders, and map
the checked resume binder to Buslane resume values with the correct function
shape. Remove the temporary handler-lowering abort.

## Acceptance Criteria
- [ ] Checked Lane handlers lower to Buslane `Handle` expressions.
- [ ] Handler tables are grouped by singleton handled effect.
- [ ] Operation alternatives bind positional payload values, not source
  payload patterns.
- [ ] Resume binders lower to Buslane resume values with the checked resume
  function shape.
- [ ] The temporary handler-lowering abort is removed.
- [ ] Focused integration snapshots cover direct arms, payload-pattern arms,
  final clauses, and resume usage.
- [ ] `moon test modules/lanec/elaborate --warn-list +73` passes.
- [ ] `moon check modules/lanec --warn-list +73` passes.
- [ ] `moon info && moon fmt` has been run after the change.
