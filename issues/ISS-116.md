# ISS-116: Warn on unused private top-level declarations

## Description
Lane should emit a warning when a private top-level value, function, type, effect, or alias declaration is never used by the current module. Public declarations are exported API surface and should not be warned merely because they are unused internally.

## Design
Treat this as a declaration-reachability lint over resolved semantic symbols, separate from local unused binding checks. The analysis should respect exports, imports, offers, generated entry expectations, and cross-declaration references inside the same module. It should not require whole-program linking or file-system module discovery.

## Acceptance Criteria
- [ ] Unused private top-level declarations produce warning diagnostics at declaration locations.
- [ ] Public declarations do not warn solely because they are unused internally.
- [ ] Private declarations referenced by other declarations in the module do not warn.
- [ ] The behavior is locked by valid/invalid or diagnostic fixtures.
- [ ] CLI and LSP diagnostics render the warning consistently.
