## Description
Add the execution-local Host Object Table, generational handles, owned host results, finalizers, resource limits, and erased host-call value model required by `Opaque`.

## Design
Store each host-language type in its own typed `HostObjectStore[T]`. Execution-local table entries retain only typed-store keys and release closures, while Lane ARC wrappers retain backend-private table handles. Parameters use typed store lookup; results transfer one independently owned value-finalizer pair through atomic adoption. Finalizers receive no Runtime Context, are non-observable and non-retriable, and may panic fatally.

## Acceptance Criteria
- [x] `ExecutionConfig.max_host_objects` has consistent disabled, bounded, and invalid-value semantics.
- [x] Handles validate execution identity, slot index, and generation.
- [x] Final release removes the entry and invokes its finalizer exactly once.
- [x] Partial result adoption finalizes the pending payload without leaking a slot.
- [x] Host SDK exposes erased core values and typed registration adapters without unchecked payload casts.
- [x] Focused lifecycle, aliasing, limit, stale-handle, and panic tests pass.
