## Description
Define the compiler-owned semantic completion data model and expose a position-specific query API in `lanec`.

## Design
Add `CompletionTrigger`, `CompletionKind`, `CompletionEntry`, and a completion query entry point that accepts source inputs, source identity, position, and trigger context. Keep the query separate from ordinary `AnalysisResult`; do not precompute completion scopes for every analysis request. Completion entries should carry labels, semantic kinds, optional detail/documentation, optional plain insert text, replacement range, and sort text without depending on LSP types.

## Acceptance Criteria
- [ ] `lanec` has public completion model types suitable for LSP and future tools.
- [ ] A position-specific completion query can be called with the same root/library input shape used by analysis.
- [ ] V1 does not introduce snippet insert text or LSP-specific fields in compiler types.
- [ ] Unit tests cover empty/unsupported completion queries and basic model serialization or debug output where appropriate.
