## Description
`GenericApplicationEvidence` centralizes generic inference inputs, but its interface is still shallow: callers must know how to fill parameter templates, result templates, effect templates, order flags, diagnostic labels, and reporting flags. This leaks generic inference implementation details into call typing, variant typing, operation typing, and contextual calls.

## Design
Provide semantic entrypoints for the real use cases, such as callable instantiation, variant owner inference, variant hidden witness inference, operation owner inference, and operation hidden witness inference. Keep the low-level evidence record private to the implementation so callers choose a domain operation instead of assembling inference internals.
