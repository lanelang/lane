## Description
Use LoisVM consuming data and environment projection instructions when the source owner dies at the projection and selected members must become owned results.

## Design
Select consuming projections in VM CFG before ARC insertion, using final local ownership and liveness facts. Group selected fields or captures from the same owner into one ordered projection. Leave borrowing projections in place when the owner remains live or the result is only block-local.

## Acceptance Criteria
- [x] Owned product accessors can lower to `consume_fields` instead of borrow-retain-release sequences.
- [x] Nonescaping environment destruction can use `consume_captures` where applicable.
- [x] Multiple selected members use one ordered consuming projection.
- [x] Shared-object and unique-object runtime behavior remain equivalent.
- [x] Interpreter and Wasm conformance tests cover selected and discarded owned members.

## Notes
- 2026-07-17: The baseline has 22 `borrow_field` operations and zero `consume_fields`; several accessors immediately retain a field and release the source object.
- 2026-07-17: Started after ISS-224 reduced statically known callable dispatch while preserving escaping and reusable closures.
- 2026-07-17: Selection requires an owned source whose complete remaining use set is the consecutive projection group, plus at least one projected borrowed result with a consuming use. Block-local read-only projections remain borrowed.

## Close Notes
Added pre-ARC consuming projection selection for both data fields and closure captures. Consecutive projections from the same owned source are grouped into one ordered instruction when no dependent borrow crosses the consumption point and a selected borrowed result must become owned; prior tag and layout-witness reads are allowed because they materialize independent trivial values. Result metadata is promoted to `Owned`, allowing ARC finalization to transfer members without retain-copying them. Focused tests cover returned owned fields, grouped captures, preservation of block-local borrows, and final bytecode without retain/release; existing interpreter host-object and Wasm descriptor tests cover shared owners and discarded members. The Coroutine Scheduler now emits 10 `borrow_field`, eight `consume_fields`, zero `borrow_capture`, and 100 `consume_captures`; explicit retain/release instructions total 45. Verification: VM CFG 19/19, LoisVM lowering 35/35, interpreter 21/21, Wasm 45/45, all runnable examples, and `git diff --check`.
