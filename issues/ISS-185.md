## Description
Lower matches on nominal data types with exactly one constructor as irrefutable in-block projections instead of emitting a `SwitchTag` and branch CFG.

## Design
A verified value of a single-constructor nominal type cannot carry another constructor tag. Bind its match binder, hidden witnesses, and payload fields in the current block, lower the selected constructor/default body directly, then restore lexical bindings. This avoids manufacturing branch ownership transfers for dictionary projection and other product-like data.

## Acceptance Criteria
- [x] Single-constructor data matches emit no tag switch or unreachable default block.
- [x] Hidden witnesses and payload adaptation use the same projection rules as multi-constructor alternatives.
- [x] Owned values from the enclosing block remain valid in the inlined alternative.
- [x] Generated effect-dictionary projection passes VM CFG ownership finalization.
- [x] LoisVM lowering tests pass without warnings.

## Close Notes
Single-constructor nominal matches now bypass tag loading, `SwitchTag`, and unreachable default creation. A shared data-alternative binder retains the existing hidden-witness and payload projection/adaptation logic for both inlined and multi-constructor paths. The regression captures an owned local closure outside a product-like match, calls it inside the constructor alternative, verifies no switch is emitted, and executes successfully on every MoonBit target. The production runtime-effect lowering now passes VM CFG ownership finalization; its remaining assertions describe the obsolete pre-CPS function-table shape and are updated by ISS-166.
