# ISS-128: Move typecheck diagnostics onto typed ownership

## Metadata
- Type: task
- Status: closed
- Priority: 1

- Assignee: codex

- Updated: 2026-07-06
- External ref: none

## Description
Typecheck diagnostics currently depend on `lanec/compile/diagnostic.mbt` for user wording, even though the typechecker owns the semantic facts and type/effect display context. This keeps the largest and most volatile diagnostic switch in compile.

## Design
Move typecheck diagnostic conversion into the typecheck package, backed by the shared typed diagnostic API. Keep display helpers owner-local. Preserve recent improvements for type mismatch, generic inference, contextual offers, pattern coverage, effect mismatch, and unused local warnings.

## Acceptance Criteria
- [x] `TypeCheckDiagnostic` implements or delegates to the shared typed diagnostic API.
- [x] `lanec/compile` no longer contains the typecheck diagnostic wording switch.
- [x] Recent generic inference diagnostics still render as direct expected/found mismatches where appropriate.
- [x] Pattern coverage and contextual offer diagnostics keep their secondary facts and suggestions.
- [x] `moon test modules/lanec/typecheck --warn-list +73` and `moon test modules/lanec/compile --warn-list +73` pass.
