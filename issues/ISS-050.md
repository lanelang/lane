## Description
Checked Lane operation-call expressions must lower to Buslane `perform`
expressions, not ordinary value calls or source-shaped placeholders.

## Design
Use the operation metadata mapping from ISS-048 and the effect/type lowering
from ISS-049 to lower checked operation calls into Buslane `Perform` nodes.
Reuse the existing expression lowering order for payload expressions so
evaluation order and expression effects remain explicit in Buslane.

## Acceptance Criteria
- [x] Checked Lane operation-call expressions lower to Buslane `perform`.
- [x] Payload expressions lower in the same left-to-right order as ordinary
  call arguments.
- [x] Operation identity is resolved through the lowering metadata mapping.
- [x] Operation calls are not lowered as ordinary function calls.
- [x] Focused integration snapshots cover Lane operation calls through Buslane
  `perform`.
