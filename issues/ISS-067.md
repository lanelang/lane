## Description
The diagnostic core already has `Suggestion`, `TextEdit`, and `Applicability`, but the CLI renderer and compiler stages do not yet make suggestions a first-class user experience. Rust-style diagnostics should show precise edits when the compiler has enough confidence.

## Design
Render suggestions as concrete source replacements or insertion snippets in the CLI, and map the same edits into LSP code actions where applicable. Only mark suggestions machine-applicable when the compiler can guarantee the edit is local and correct.

## Acceptance Criteria
- [x] CLI rendering shows suggestion edits in a readable form with applicability.
- [x] LSP code actions are generated from the same `Suggestion` data.
- [x] At least three diagnostics produce suggestions, including one machine-applicable fix and one maybe-incorrect fix.
- [x] Tests cover both rendered text and LSP JSON/code-action transport.
