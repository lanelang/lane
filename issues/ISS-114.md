# ISS-114: Move empty bracket parse diagnostics into grammar context

## Description
ISS-112 introduced targeted diagnostics for empty generic argument lists and empty type parameter lists. The behavior is correct, but the implementation classifies the error in the parser wrapper by looking backward in the token stream and inspecting rendered expected syntax. That is fragile: the parser wrapper should not infer grammar context from neighboring tokens.

## Design
Move the empty bracket list classification into the yacc grammar. Specific grammar productions that recognize empty bracket syntax should raise a generated parse error marked with an explicit diagnostic context, and the parser wrapper should translate that context into the public `ParseErrorKind`. Keep the user-facing diagnostics introduced by ISS-112 unchanged.
