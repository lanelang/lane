## Description
Complete the compiler pipeline that lowers Buslane effect core into effect-erased input for LoisVM. The current LoisVM execution core is implemented, but the lowering entry only rewrites linked primitive runtime operations and rejects residual `handle`, `resume`, non-runtime `perform`, and generic runtime operations.

## Design
First elaborate Buslane handler syntax into a compiler-private handler runtime protocol, then implement the documented `mon-trans`, `open-resolve`, and `monadic-lift` sequence before ordinary ANF and VM CFG lowering. Preserve Lane deep-handler and multi-shot resume semantics while representing lowered continuations and handler contexts as ordinary values and closures managed by LoisVM ARC. LoisVM bytecode must remain free of effect-specific instructions and dispatch structures. The synchronous primitive-only v1 host ABI and the decision not to verify trusted bytecode remain non-goals for this epic.

## Acceptance Criteria
- [x] The three lowering passes have explicit, tested input and output contracts.
- [x] Lane programs containing lexical handlers and resume expressions lower to effect-erased LoisVM bytecode.
- [x] Residual outer runtime effects lower to ordinary runtime-import calls after source handlers are resolved.
- [x] No LoisVM bytecode instruction or runtime object encodes `perform`, `handle`, `resume`, or effect dispatch.
- [x] Interpreter and Wasm execution agree on the effect-lowering regression corpus.
- [x] Documentation reflects the implemented pipeline rather than the temporary primitive-effect rewrite.

## Close Notes
Completed through ISS-162 through ISS-168 and the implementation follow-ups ISS-173 through ISS-185. The production `lower_linked_program_to_bytecode_with_runtime_effects` path installs validated root runtime handlers, elaborates Buslane handlers, runs mon-trans and selective CPS ABI rewriting, resolves open contexts, lifts monadic continuations to ordinary Buslane, and only then enters ANF, VM CFG ownership/ARC insertion, and bytecode emission. ADR-0067 and the lanec glossary document each pass contract and the implemented dictionary-CPS protocol. The LoisVM bytecode, interpreter, runtime, and Wasm layers contain no effect-specific instruction, object, handler stack, or dispatch table; residual host operations are ordinary unified-function-table `RuntimeImport` calls. The native conformance corpus compares complete interpreter and Wasmoon traces for direct and nested handlers, nearest-handler dispatch, final and arm effects, captured-String single/multi-shot resume, generic layout witnesses, and the primitive host boundary. Final verification: `moon info && moon fmt` with no interface changes; `moon check --target all --warn-list +73`; `moon test modules/lane --target native` (48/48); `moon test --target native` (697/697); `git diff --check`.
