# ISS-057: Consolidate typecheck application inference

## Description
The typechecker has no deep seam for application checking. Ordinary calls,
contextual calls, checked-expression calls, function literals, effect operation
calls, and nominal constructor calls each carry pieces of the same policy:
arity validation, named or auto argument handling, generic type argument
inference, expected-result collection, checked argument construction, effect
collection, and `CannotInferExpression` diagnostics.

This makes future source features risky because a new callable form can update
one path while leaving another path with different inference or diagnostic
behavior. The recent structured inference diagnostic work also showed that
adding one semantic cause requires touching many call and function-literal
sites instead of one application layer.

## Design
Introduce a deeper application-checking interface in the typechecker. The
interface should accept a callable shape, source arguments, an optional expected
result, and the desired checking mode, then return an instantiated callable
signature plus checked arguments/effects or a single structured diagnostic path.

Adapters for ordinary function calls, contextual calls, operation calls, and
constructors should describe their callable-specific inputs without reimplementing
the shared inference and arity policy.

Non-goals:
- Do not redesign the whole typechecker in this issue.
- Do not change user-visible diagnostics unless the new shared path exposes
  intentional improvements that are covered by snapshots.

## Acceptance Criteria
- [ ] Ordinary calls, contextual calls, checked-expression calls, effect operation calls, and constructor calls share one application/inference policy seam.
- [ ] Type argument inference and expected-result collection are not duplicated across call-like forms.
- [ ] `CannotInferExpression` reason selection for call-like forms is centralized.
- [ ] Regression tests cover at least one generic function call, contextual call, effect operation call, and generic constructor call through the shared path.
- [ ] `moon test modules/lanec/typecheck --warn-list +73` passes.
- [ ] `moon test modules/lanec/compile --warn-list +73` passes.
- [ ] `moon info && moon fmt` has been run after the change.
