## Description
`examples/valid/14_existential_enums.lane` is a valid existential enum example, but running the `packed_int` entry currently reports invalid Buslane match/type-binder diagnostics instead of a valid packed value.

Observed command:

```sh
moon -q run --target native modules/lane -- run examples/valid/14_existential_enums.lane --entry packed_int --stdlib ../stdlib
```

Observed output:

```text
invalid_program {
  invalid_alternative_binder_count
  unknown_type_parameter tparam#23
}
```

## Design
Investigate lowering for enum variants with hidden type parameters and match alternatives that open existential binders. The generated Buslane alternative binder list and hidden witness substitution likely disagree with data-constructor metadata. Keep the fix in compiler lowering or metadata generation; do not relax the Buslane verifier's binder-count checks.
