# ISS-152: Remove derived contextual offer forwarding

## Description
Contextual resolution currently recursively derives additional offers from fields marked as offered. After removing field offer syntax, the typechecker should consume only a flat set of explicit named offers and offered parameters.

## Design
Delete recursive forwarding and field-path candidate support. A contextual offer is only an explicit `offer name : T = expr` binding or an `offer` / `auto offer` parameter. Ambiguity remains an error when two explicit visible offers have the same required type.

## Acceptance Criteria
- [x] `offer_forwarding_fields` and field-path candidate machinery are removed.
- [x] Contextual offer diagnostics name only explicit offer bindings or offered parameters.
- [x] Having only `Compare[Int]` available does not satisfy a required `Equal[Int]`.
- [x] `auto` parameters do not become offers unless explicitly marked `offer`.
