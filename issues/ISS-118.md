# ISS-118: Warn on unused imports and opens

## Description
Lane should emit a warning when an imported module or open exposure is not used by the module being compiled.

## Design
Base the warning on resolved name usage and import/open exposure records rather than raw text. Imports used only for type names, effect names, constructors, operations, or qualified value references must count as used. Open exposure should warn only when it contributes no selected name to resolution. This issue should stay separate from missing-import errors and module-input validation.

## Acceptance Criteria
- [ ] Completely unused imports produce warning diagnostics at import locations.
- [ ] Imports used through values, types, effects, constructors, operations, or qualified references do not warn.
- [ ] Open exposures that contribute no resolved names produce warning diagnostics.
- [ ] Open exposures that contribute at least one resolved name do not warn.
- [ ] The behavior is locked by focused tests and CLI/LSP diagnostic fixtures.
