## Description
Introduce the explicit compiler optimization seam that is currently missing between effect lowering and ANF lowering.

## Design
Create a coherent `lanec` core-optimization package over Buslane programs. Run effect-sensitive rewrites after monadic continuation lifting and before residual effect erasure, while effect rows still distinguish observable calls. The optimizer accepts explicit entry and initializer roots and preserves the compiler-private lowered-core contract; optimizer-local occurrence facts are not persisted. Ordinary Buslane verification remains after residual effect erasure because intermediate monadic-lift output intentionally contains temporary type scopes that the public Buslane verifier does not model.

Keep administrative post-erasure cleanup separate from semantic rewrites that require effect information.

## Acceptance Criteria
- [x] The executable pipeline has an explicit effect-aware core-optimization phase before effect erasure.
- [x] The optimizer interface receives roots and hides pass ordering and fixpoint mechanics.
- [x] Rewrites cannot classify an effectful expression as removable after effect information is erased.
- [x] The optimizer validates explicit roots, and post-erasure executable tests verify the resulting ordinary Buslane program.
- [x] Explore exposes the optimized core stage without hard-splitting unrelated lowering passes.
