## Description
Buslane text import/export must be able to represent effect-core programs so
fixtures, snapshots, and future Lane IR explorer tooling can roundtrip them.

## Design
Add Buslane expression forms for `perform`, handler tables, operation
alternatives, and resume values, then update pretty printing and text parsing.
Handler tables remain grouped by singleton effect before operation
alternatives. Operation alternatives bind evaluated payload values
positionally; source payload patterns are lowered before Buslane.

## Acceptance Criteria
- [ ] Buslane expressions include `perform`, handler tables, operation
  alternatives, and resume values.
- [ ] Pretty output covers effect metadata, function latent effects, `perform`,
  handlers, and resume values.
- [ ] Text parsing and import/export support all effect-core forms.
- [ ] Roundtrip tests cover pure function types, open effect rows, `perform`,
  and deep handlers.
- [ ] `moon test modules/buslane/text` passes.
- [ ] `moon info && moon fmt` has been run after the change.
