## Description
Add a reusable `bytecodec` MoonBit module for strict byte-level binary encoding and decoding. This module is the primitive foundation for Buslane binary codecs and Lane artifact binary payloads.

## Design
Create `modules/bytecodec` with a small public API:

- `ByteWriter` for appending bytes and little-endian primitive values.
- `ByteReader` for reading from a byte view or bytes value with offset tracking.
- primitive codecs for `u8`, `u32le`, signed integer fields where needed, booleans, length-prefixed UTF-8 strings, and length-prefixed raw bytes.
- typed decode `suberror` carrying offset and expected/invalid-value details.
- `expect_end` support so callers can reject trailing bytes.

MoonBit bitstring patterns may be used inside `bytecodec` for primitive or fixed-header decoding, but higher-level artifact and Buslane schema decoders should use the reader/writer API.

Non-goals: Lane artifact schema, Buslane AST encoding, generic reflection-based serialization, varints, compression, or checksum policy.

## Acceptance Criteria
- [x] `modules/bytecodec` exists as a standalone module/package suitable for reuse.
- [x] The API covers fixed-width little-endian primitive writing and reading.
- [x] Decode failures use typed `suberror` rather than `String` or accumulated arrays.
- [x] Tests cover endian behavior, bounds errors, invalid UTF-8, string/bytes length handling, and trailing byte rejection.
- [x] `moon info && moon fmt` and relevant tests pass.
