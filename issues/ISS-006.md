## Description
Create a queryable semantic index from resolved and checked source data so editor features can map source positions to definitions, references, and hover payloads.

## Design
The index should collect declaration locations for value, type, field, variant, type parameter, and type-member symbols. It should also collect reference ranges from checked or resolved expressions and patterns. Use checked types and kinds for hover text where available. Avoid exposing raw backing arrays as the public query API.

## Acceptance Criteria
- [ ] Analysis results include a definition index keyed by compiler symbol identity.
- [ ] Analysis results include source-range reference entries that point to symbol identities.
- [ ] Analysis results include hover payloads for supported declarations and references.
- [ ] Query helpers can find the smallest semantic range containing a source position.
- [ ] Unit tests cover local definitions, top-level definitions, fields, variants, and type names.
