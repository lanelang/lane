## Description
The resolver diagnostic kind `PayloadlessVariantCall` and pretty text `payloadless_variant_call` still carry the old meaning that calling a payloadless variant is invalid. The current rule is the opposite: payloadless variants are constructors and must be written with empty parentheses.

## Design
Rename the diagnostic kind and internal pretty output to express the current rule, for example `PayloadlessVariantWithoutParens`. Update compile diagnostic conversion and tests accordingly. Keep the public user-facing diagnostic code stable unless there is a strong reason to change it.

## Acceptance Criteria
- [x] Resolver diagnostic names no longer imply that `none()` is invalid.
- [x] Resolver pretty/debug output uses without-parentheses terminology.
- [x] Compile diagnostics still render the current user-facing message and help.
- [x] Resolve and compile tests pass with `--warn-list +73`.
