## Description
Add ANF-local cleanup for administrative bindings introduced by normalization after semantic core optimization has completed.

## Design
Keep the pass inside the `lanec/anf` package. Perform local copy propagation, trivial-let removal, dead pure temporary removal, constant propagation, and safe local common-subexpression reuse. Do not reimplement whole-program reachability, generic specialization, or effect-sensitive inlining at ANF.

## Acceptance Criteria
- [x] Single-use administrative aliases are eliminated.
- [x] Dead pure temporaries and duplicate local constants are removed.
- [x] Evaluation order of calls, allocations, and observable operations remains unchanged.
- [x] ANF output remains well scoped and accepted by LoisVM lowering.
- [x] Focused snapshots show reduced temporary counts.

## Close Notes
Added a package-private ANF rewrite scope and integrated cleanup into Buslane-to-ANF lowering. The pass recursively propagates atomic value/temp aliases through functions, matches, handlers, and recursive scopes while masking binders correctly; it does not remove or reorder calls, allocations, type applications, performs, or handlers. Focused structured snapshots reduce a four-binding alias chain to one call binding, and the Coroutine Scheduler report contains zero `let value = temp` aliases with 117 required call/allocation temporaries. Verification: ANF 4/4, LoisVM lowering 35/35, interpreter 21/21, Wasm 45/45, Lane 56/56, all runnable examples, warning-enabled check, and `git diff --check`.
