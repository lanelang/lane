# ISS-065: Make module import diagnostics explain actual inputs

## Description
When an import such as `Builtins` is missing, the diagnostic currently says no matching module input or interface artifact exists. If the user already passed an interface artifact such as `builtins.lmi`, this is not enough: the report should explain what was passed, what module names those artifacts declare, and why none matched the requested import.

## Design
Preserve enough module input and artifact metadata during module input checking to report the requested module, candidate inputs, candidate interface module names, and the exact mismatch. Do not search the file system by module name; explain the explicit-input model clearly.

## Acceptance Criteria
- [x] Missing-import diagnostics include the imported module name and the explicit module/interface inputs considered.
- [x] Interface artifacts with non-matching embedded module names are reported as candidates, not silently ignored.
- [x] The help text recommends a concrete next action based on the mismatch.
- [x] CLI golden tests cover passing an `.lmi` whose filename looks plausible but whose embedded module identity does not match the import.
