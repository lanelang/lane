## Description
Implement the agreed run/runtime convention for executable Lane entries that use explicit, user-provided `Stdlib` artifacts. The convention must not implicitly inject Stdlib, must not pin an official Stdlib fingerprint, and must validate runtime-handled effects from source-level exported module/effect/operation/signature metadata at the execution boundary.

## Design
Track this as an execution-boundary feature, not a language-level `main` rule. `Stdlib` is a conventional module path whose exported shapes may be supplied by any user artifact. `lane run` and `lane runobj` should execute only zero-argument `Unit` entries whose fully expanded closed concrete effect set is empty or covered by registered runtime conventions. The initial command convention handles only `Stdlib.Io.Write.println(String) -> Unit`.

## Acceptance Criteria
- [x] `../stdlib` provides the initial `Stdlib` effect and alias shapes.
- [x] Linked artifacts expose enough source-level exported effect metadata for runtime convention validation.
- [x] `lanec`/Buslane runtime APIs can validate and invoke outer runtime effect handlers.
- [x] `lane run` and `lane runobj` execute only valid executable entries and do not debug-render arbitrary public values.
- [x] ADR-0055 and runtime glossary remain aligned with the implementation.
