## Description
Implement the accepted host-boundary design for user-declared External Effects and zero-parameter External Types, including their erased `Opaque` runtime ABI, owned host-object lifecycle, interpreter behavior, Wasm behavior, and typed host registration surface.

## Design
Use `extern type E : Effect` for non-handleable observable host dependencies and `extern type T : Type` for opaque nominal host objects. Preserve their semantic flavors through checking, Buslane, and linking, then erase External Types to one direct `Opaque` ABI kind during LoisVM lowering. Keep host objects in an execution-local generational table with Lane-owned ARC wrappers, borrowed parameters, owned results, and per-instance finalizers.

## Close Notes
Completed through ISS-206 to ISS-215. Lane now supports user-declared External Effects and zero-parameter External Types, preserves their flavors through Buslane and linking, erases direct External Type parameters and results to `Opaque`, and serializes complete Runtime Import signatures. The shared runtime owns execution-local generational Host Object Tables, typed host registration, borrowed arguments, atomic owned-result adoption, ARC wrappers, explicit cleanup, finalization, and host-object limits.

The interpreter and Wasmoon paths execute the same source-produced image with matching output, ownership, failure, and cleanup behavior. Executable entries accept closed rows containing only External Effects, and lowering guarantees the canonical Reference layout for reachable `Opaque` imports. Public README, canonical contexts, ADRs, and valid/invalid examples match the implementation.

Final verification: `moon info && moon fmt`; `moon check --target all -d`; `moon test` (5 wasm, 633 wasm-gc, 193 native); `tools/check-lane-run-examples.sh`; focused README and External Type cross-backend tests; `git diff --check`.
