## Description
Reduce Wasm-specific function-table and per-function code-generation overhead after high-level and VM optimizations have exposed the final callable set.

## Design
Include only address-taken Lane functions and required helper functions in the indirect-call table, with deterministic remapping for packed callable values. Select lighter entry/cleanup scaffolding for functions that provably need no owned cleanup while preserving fatal unwind, logical call depth, and runtime-limit semantics. Apply only local Wasm instruction cleanup; do not duplicate core or CFG optimization.

## Acceptance Criteria
- [x] Direct-only functions are absent from the indirect-call table.
- [x] Packed callable function indices are remapped deterministically.
- [x] Functions without owned cleanup use a smaller correct Wasm path where provable.
- [x] Tail calls, fatal unwind, runtime limits, and host imports retain existing behavior.
- [x] Generated Wasm validates and executes in Wasmoon interpreter and JIT modes.

- 2026-07-17: The final Scheduler module emits 174 Wasm functions, a 141-slot table containing 140 initialized callable entries, and 406,092 bytes of WAT.

## Close Notes
Added a deterministic callable-table plan that scans materialized function and closure values, maps only those bytecode function IDs to dense one-based table indices, and appends required helper functions after the Lane callable region. Direct calls retain physical Wasm indices while packed callable values use the compact mapping. Functions with only trivial slots now omit per-slot liveness locals and frame cleanup while retaining call-depth checks, fatal unwind, tail calls, runtime limits, and host-import adapters. The Scheduler table falls from 212 to 141 slots, Wasm functions from 221 to 174, and generated WAT from about 604 KiB to 406,092 bytes. Verification: compiler 3/3, Wasm 45/45, all runnable examples, and identical Wasmoon JIT/interpreter output `1112211322`.
