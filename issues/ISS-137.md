# ISS-137: Remove whole-checker silent retyping from handler residual analysis

## Description
Handler residual effect computation currently copies the full `SemanticChecker` and re-checks handler arm bodies silently. This makes a local effect measurement depend on copying declarations, values, offers, diagnostics, and expected effect state, which is an expensive and fragile seam.

## Design
Replace full-checker silent retyping with a narrower effect summary or typed arm planning interface. The handler checker should compute residual effects without duplicating the entire checker state and without relying on diagnostics length or copied mutable context as control flow.
