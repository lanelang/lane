## Description
Handler operation arms and the final branch must agree on the handler result
type. Resume binders must receive function types whose parameter is the
operation result type and whose latent effect is the handler residual effect.

## Design
Extend handler checking with the result-type and residual-effect judgment. Check
the handled expression effect, remove handled concrete effects from the
residual set, type final and operation-arm bodies against a common result type,
and bind resume continuations as deep-handler resume functions.

## Acceptance Criteria
- [ ] Final branch body and operation-arm bodies are checked against a common
  handler result type.
- [ ] Resume binders receive `(Q) -> R ! E_res` types, where `Q` is the
  operation result type, `R` is the handler result type, and `E_res` is the
  residual handler effect.
- [ ] Handled effects are removed from the handled expression effect when
  computing the handler expression effect.
- [ ] Operation arm direct effects contribute to the residual effect according
  to the current Lane effect judgment.
- [ ] Tests cover resume typing, final/body result mismatches, and unhandled
  residual effects.
- [ ] `moon check modules/lanec --warn-list +73` passes.
- [ ] `moon info && moon fmt` has been run after the change.
