## Description
The current renderer can print a header, one or more source snippets, and simple notes or helps. It still lacks the layout behavior users expect from a mature compiler: grouped labels, related locations, better multi-line spans, tab and Unicode display handling, smart truncation, and readable suggestion blocks.

## Design
Keep rendering independent from compiler meaning. The renderer should consume structured diagnostics and produce terminal text with stable wrapping, source grouping, clear primary and secondary labels, related information sections, and suggestion edit blocks.

## Acceptance Criteria
- [x] Multi-label diagnostics are grouped by source and ordered predictably.
- [x] Related diagnostics and suggestion edits render in CLI output.
- [x] Unicode width, tabs, long lines, and multi-line spans have documented behavior and regression tests.
- [x] Renderer tests cover plain output without depending on terminal color.
