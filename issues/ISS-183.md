## Description
Prevent selective CPS metadata rewriting from turning nominal data parameters and hidden existential binders of kind `Effect` into kind-`Type` companion parameters that require runtime layout witnesses.

## Design
Companion parameters represent explicit effect contexts only at polymorphic callable boundaries. Effect arguments embedded in nominal data applications, constructor witnesses, and pattern binders remain compile-time-only `EffectArgument` values so the existing LoisVM data lowering erases them without stored witnesses.

## Acceptance Criteria
- [x] Nominal type parameters and hidden constructor parameters of kind `Effect` retain their original identity and kind.
- [x] Constructor effect arguments and hidden witnesses remain `EffectArgument` values after mon-trans.
- [x] Pattern-bound effect existentials remain compile-time-only and require no layout witness.
- [x] Callable effect parameters still lower to ordinary companion context parameters.
- [x] Effect-lowering tests pass on all targets without warnings.

## Close Notes
Restricted companion rewriting to callable effect-context boundaries. Nominal type parameters, hidden constructor parameters, constructor effect arguments, and pattern binders now preserve their original kind-`Effect` identities, while dictionary context arguments still become ordinary companion types. Mon-trans validation explicitly permits these erased nominal-data arguments but continues rejecting residual callable effect parameters and type applications. Regression coverage verifies the full effect pipeline and production LoisVM lowering retain zero runtime witnesses.
