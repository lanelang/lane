# ISS-123: Remove obsolete inference diagnostic catch-all

## Metadata
- Type: task
- Status: closed

## Description
After known inference failure families are migrated, the legacy `CannotInferExpression` path should be removed so new code cannot continue adding vague diagnostics.

## Design
Delete or sharply restrict `CannotInferExpression`, `CannotInferReason`, `cannot_infer_diagnostic`, and `report_application_inference_failure`. Any remaining truly unexpected case should use an internal diagnostic shape, not a normal user-facing inference error.

## Acceptance Criteria
- [x] No normal typecheck path emits `CannotInferExpression`.
- [x] No compile diagnostic path renders the generic "cannot infer expression type" catch-all for known cases.
- [x] The diagnostic golden corpus covers all migrated inference failure families.
- [x] Stale fixture expectations for `error[E3010]: cannot infer expression type` are removed or limited to an explicitly justified internal fallback.
- [x] Full `moon test --warn-list +73 --diagnostic-limit 200` and `tools/check-lane-run-examples.sh` pass.
