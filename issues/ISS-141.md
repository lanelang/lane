## Description
Lane `.lmi`, `.lmo`, and `.lbp` files currently use a binary wrapper around UTF-8 artifact text. The artifact format should become a real binary format: a versioned Lane artifact container with structured fixed-order binary payloads. The production compile, link, exec, and inspect paths must stop depending on artifact text parsing.

## Design
Use a layered design:

- `modules/bytecodec` provides strict byte-level readers, writers, fixed-width little-endian primitive codecs, length-prefixed strings/bytes, offset tracking, and typed decode errors.
- `modules/buslane/codec` owns structured binary encoding and decoding for Buslane programs, metadata, types, expressions, and identities.
- `modules/lanec/compile` owns artifact container and artifact payload schemas, delegates Buslane sections to `buslane/codec`, and removes artifact text parsing from the production load path.

Container version, artifact schema version, and Buslane codec version are separate compatibility boundaries. Decoders reject trailing bytes at top-level and section boundaries. `inspect` remains the human-readable semantic projection and should render summaries plus Buslane pretty output, not artifact text.

## Acceptance Criteria
- [x] `bytecodec`, `buslane/codec`, and lanec artifact binary payload work are tracked by child issues.
- [x] Production artifact load paths no longer parse UTF-8 artifact text.
- [x] Artifact text parser tests are removed or replaced with binary/inspect tests.
- [x] Documentation records the binary artifact contract and its non-goals.
