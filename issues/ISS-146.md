## Description
`modules/lanec/elaborate/buslane_lower_*.mbt` is a large, internally cohesive
lowering stage from checked Lane programs to Buslane. It has an explicit
lowerer, registry, collection phase, import seeding, expression/pattern lowering,
handler lowering, and symbol allocation. The file family is already behaving
like a package, but it is still mixed into `lanec/elaborate`.

## Design
Evaluate and implement a package split only if the dependency direction stays
clean. The new package should own the lowering implementation and expose a small
facade for checked-source-to-Buslane lowering. Shared public data that would
otherwise create an `elaborate <-> buslane_lowering` cycle must be moved to an
appropriate common package rather than duplicated or hidden behind compatibility
wrappers.

Non-goals:
- Do not change Buslane IR semantics.
- Do not rewrite lowering algorithms solely because files move.
- Do not keep legacy forwarding APIs unless a current downstream caller truly requires them.

## Acceptance Criteria
- [x] Buslane lowering has an explicit package boundary or a documented reason why the split should be deferred.
- [x] No cyclic package dependency is introduced.
- [x] The lowering facade remains small and understandable from package interfaces.
