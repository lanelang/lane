## Description
`resolve/collect.mbt` is approximately 3,700 lines and owns declaration precollection, import publication, type and effect resolution, expression and pattern resolution, provenance recording, lookup helpers, and diagnostics. These responsibilities share mutable resolver state but do not form one cohesive implementation unit. Changes to one namespace or syntax family repeatedly touch the same large file and conceal duplicated conventions.

## Design
Mechanically split the existing implementation into same-package files along its natural responsibilities: declaration/import collection, type/effect resolution, expression/operation resolution, pattern resolution, and lookup/provenance support. Keep `Resolver` state and invariants private to the package, preserve dependency direction, and avoid introducing compatibility facades or new packages with one consumer. Resolve concrete duplicated conventions while moving them rather than copying helpers into each file.

## Acceptance Criteria
- [ ] Resolver declaration/import, type/effect, expression/operation, pattern, and lookup/provenance implementations have clear same-package ownership.
- [ ] `collect.mbt` is removed or reduced to orchestration rather than remaining the implementation sink.
- [ ] No helper implementation is duplicated across the resulting files.
- [ ] Public resolver interfaces and generated `.mbti` remain unchanged unless a separately justified semantic fix requires otherwise.
- [ ] Existing resolver, typecheck, analysis, compile, formatter, and examples behavior remains unchanged.
- [ ] `moon info`, `moon fmt`, focused tests, full tests, examples, and `git diff --check` pass.
