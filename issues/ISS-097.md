## Description
Hard switch Lane compiler and formatter diagnostics to a single source-aware diagnostic path. Diagnostic output is part of the compiler interface: source diagnostics must be accurate, actionable, and rendered with source text and spans rather than compact pretty-printed fallback strings.

## Design
Introduce `modules/lanec/diagnostic` as the compiler-owned diagnostic package. Keep `modules/diagnostic` as generic infrastructure with no Lane semantics. Move compiler-facing diagnostic data types into `lanec/diagnostic`, make both compile and format produce the same `CompilerDiagnostic` model, centralize conversion to `diagnostic/core.Diagnostic`, and delete legacy `Pretty`/compact diagnostic paths. Command, artifact, runtime, and Buslane verifier reports stay outside the source-diagnostic path unless they carry an explicit source origin map.

## Acceptance Criteria
- [ ] `modules/lanec/diagnostic` owns the compiler diagnostic data model and source adapter.
- [ ] `lanec/compile` and `lanec/format` produce the same `CompilerDiagnostic` type.
- [ ] CLI and LSP share the same compiler diagnostic adapter instead of duplicating conversion logic.
- [ ] User-visible source diagnostics for check, compile, run, fmt, and lsp are rendered through source-aware diagnostic infrastructure.
- [ ] Legacy diagnostic pretty/fallback output paths are removed.
- [ ] Existing command, artifact, runtime, and Buslane verifier reports remain distinct from source diagnostics.
