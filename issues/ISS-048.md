# ISS-048: Lower Lane effect metadata to Buslane metadata

## Description
Lane checked programs now carry effect declarations and operation signatures.
Buslane lowering needs to turn that semantic metadata into Buslane `EffectId`,
`OperationId`, `EffectInfo`, and `OperationInfo` entries before expressions can
refer to operations.

## Design
Extend the Buslane lowering context with a clear mapping from checked Lane
effect and operation identities to Buslane identities. Lower effect constructors
and operation signatures into Buslane metadata in the same phase that lowers
other module metadata. Keep Lane source spans, presentation names, and resolver
details outside Buslane nodes.

## Acceptance Criteria
- [x] Checked Lane effect declarations lower to Buslane effect metadata.
- [x] Checked Lane operation signatures lower to Buslane operation metadata.
- [x] Operation ownership, payload types, and result types are preserved.
- [x] Source spans and source presentation are not added to Buslane metadata.
- [x] Focused lowering snapshots or tests cover effect metadata lowering.
