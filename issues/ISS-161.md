## Description
Complete the compiler pipeline that lowers Buslane effect core into effect-erased input for LoisVM. The current LoisVM execution core is implemented, but the lowering entry only rewrites linked primitive runtime operations and rejects residual `handle`, `resume`, non-runtime `perform`, and generic runtime operations.

## Design
First elaborate Buslane handler syntax into a compiler-private handler runtime protocol, then implement the documented `mon-trans`, `open-resolve`, and `monadic-lift` sequence before ordinary ANF and VM CFG lowering. Preserve Lane deep-handler and multi-shot resume semantics while representing lowered continuations and handler contexts as ordinary values and closures managed by LoisVM ARC. LoisVM bytecode must remain free of effect-specific instructions and dispatch structures. The synchronous primitive-only v1 host ABI and the decision not to verify trusted bytecode remain non-goals for this epic.

## Acceptance Criteria
- [ ] The three lowering passes have explicit, tested input and output contracts.
- [ ] Lane programs containing lexical handlers and resume expressions lower to effect-erased LoisVM bytecode.
- [ ] Residual outer runtime effects lower to ordinary runtime-import calls after source handlers are resolved.
- [ ] No LoisVM bytecode instruction or runtime object encodes `perform`, `handle`, `resume`, or effect dispatch.
- [ ] Interpreter and Wasm execution agree on the effect-lowering regression corpus.
- [ ] Documentation reflects the implemented pipeline rather than the temporary primitive-effect rewrite.
