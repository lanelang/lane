# ISS-218: Root executable initialization from the selected entry

## Description
Executable formation currently marks every non-function top-level binding as an initializer root. This preserves unused Basic dictionaries and constants even when the selected entry cannot reach them, producing unnecessary globals, functions, allocations, and imports.

## Design
Treat the selected entry as the semantic root and close over its referenced top-level definitions. Lane top-level value initialization is pure, so an unreachable initializer has no observable execution requirement. Preserve source order among the reachable initializers and keep dependency initializers before their consumers.

The executable package owns this root policy. Occurrence analysis remains a reusable fact provider and LoisVM lowering consumes the resulting execution plan without redefining initialization semantics.

## Acceptance Criteria
- [x] Unreachable pure top-level initializers are absent from `ExecutableProgram.initializers` and `execution_roots`.
- [x] Initializers transitively referenced by the entry remain in deterministic source order.
- [x] Unused runtime imports and Basic dictionaries no longer become reachable through initializer rooting.
- [x] Interpreter and Wasm execution preserve existing observable results.
- [x] Focused executable and LoisVM lowering regressions pass.
