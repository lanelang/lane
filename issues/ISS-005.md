# ISS-005: Preserve source identity in lanec analysis inputs

## Description
Current compiler entry points mostly accept source text strings, which is enough for CLI diagnostics but insufficient for cross-file LSP locations. Add an analysis input shape that preserves a stable source identity such as URI or source ID alongside the source text.

## Design
Introduce a `ModuleInput`-style type for analysis-oriented APIs with `source_id` or URI plus text. Do not put filesystem paths into Lane module identity. Source identity is for diagnostics and LSP locations only. Keep existing simple string APIs as wrappers if needed for CLI use.

## Acceptance Criteria
- [ ] `lanec` has an analysis-oriented input representation that stores source identity and source text.
- [ ] Root and library inputs preserve source identity through parsing, import graph checking, and module compilation.
- [ ] Existing CLI check/run behavior remains unchanged.
- [ ] Tests cover at least one root plus library input retaining distinct source identities.
- [ ] `moon test modules/lanec/compile --target native --warn-list +73` passes.
