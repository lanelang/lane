## Description
Add `textDocument/definition` support to `lane lsp` using the semantic index produced by `lanec`.

## Design
Advertise `definitionProvider` in initialize capabilities after the semantic analysis API is available. Parse LSP text document position parameters, analyze the current document with workspace library overlays, query the reference at the requested position, and return an LSP `Location` for the definition. Keep URI handling in the LSP layer; do not make `lanec` own host file IO.

## Acceptance Criteria
- [ ] `initialize` advertises `definitionProvider`.
- [ ] `textDocument/definition` returns `null` for unsupported positions instead of an error.
- [ ] Same-file go-to-definition works for local and top-level value references.
- [ ] Cross-module go-to-definition works for imported public values or types when source identity is available.
- [ ] LSP tests cover request and response JSON for same-file and cross-module cases.
