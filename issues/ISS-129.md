## Description
Compile, link, and artifact diagnostics are compiler-boundary diagnostics rather than parser/resolve/typecheck diagnostics. They should stay owner-local, but they should use the same typed conversion API and should not be mixed with source-language phase wording.

## Design
Split compile/link/artifact diagnostic conversion into cohesive owner files. Keep module input, artifact schema, linking, and executable-entry errors close to the code that knows their domain. Do not move these messages into parser/typecheck/resolve packages.

## Acceptance Criteria
- [x] Compile boundary diagnostics use the shared typed conversion API.
- [x] Link and artifact diagnostics are in cohesive owner-local files.
- [x] `compile/diagnostic.mbt` no longer mixes compile/link/artifact messages with parser/resolve/typecheck messages.
- [x] `lane inspect`, `lane compile`, `lane link`, and `lane runobj` diagnostic tests pass.
- [x] Artifact parser diagnostics remain stable.
