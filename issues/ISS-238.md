## Description
Syntax already distinguishes qualified names structurally, but later paths flatten module paths and local names into dotted `String` values. Resolver, typechecker, and completion then independently split those strings to recover the structure. This loses invariants, duplicates parsing, and makes dotted-name behavior dependent on each consumer's string logic.

## Design
Introduce or reuse one semantic qualified-name representation containing a module path and local name, and preserve it across resolved references and analysis queries. Convert to display text only at presentation or serialization seams. Keep user-authored source spans attached to the structured parts. Do not parse semantic names with `split(".")` after resolution.

## Acceptance Criteria
- [ ] Qualified type, alias, owner, and completion lookups consume structured module paths and local names.
- [ ] Resolver and typechecker no longer reconstruct semantic qualified names with dotted-string splitting.
- [ ] Completion owner lookup shares the same qualified-name representation or a lossless cursor form of it.
- [ ] Diagnostics and pretty output preserve existing user-facing qualified spelling and precise spans.
- [ ] Tests cover multi-segment module paths, local names, aliases, nominal owners, and incomplete completion input.
- [ ] `moon info`, `moon fmt`, focused tests, full tests, and `git diff --check` pass.
