## Description
Diagnostic quality will regress unless examples assert the full user-facing report shape. Existing tests cover some golden diagnostics, but the next diagnostic work needs a broader corpus that includes CLI rendering, LSP transport, suggestions, related locations, and terminal-color policy.

## Design
Create small invalid Lane examples for representative syntax, resolver, module input, typecheck, effect, handler, artifact, and CLI-boundary errors. Test the structured diagnostic where possible and the rendered CLI output where human readability matters.

## Acceptance Criteria
- [x] The corpus includes examples for module import mismatch, unbound name, duplicate declaration, type mismatch, missing field, effect mismatch, handler mismatch, syntax error, and artifact error.
- [x] Golden snapshots assert rendered CLI output in plain mode.
- [x] LSP tests assert ranges, related information, and code actions for cases that support them.
- [x] The corpus documents which examples are expected to have machine-applicable suggestions.
