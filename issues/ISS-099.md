# ISS-099: Make formatter diagnostics use CompilerDiagnostic

## Description
Remove the separate formatter diagnostic model and make formatter parse/lex errors produce the shared `CompilerDiagnostic` type from `modules/lanec/diagnostic`.

## Design
Update `modules/lanec/format` APIs so `format_source` and `format_source_with_options` return `Result[String, Array[@lanec_diagnostic.CompilerDiagnostic]]`. Delete `FormatDiagnostic`, `FormatDiagnosticLabel`, `FormatDiagnosticLabelStyle`, and `Pretty for FormatDiagnostic`. Construct formatter diagnostics through the shared compiler diagnostic model. Do not make `lanec/format` depend on `lanec/compile`.

## Acceptance Criteria
- [ ] `lanec/format` depends on `modules/lanec/diagnostic`, not `modules/lanec/compile`.
- [ ] Formatter APIs return `Array[CompilerDiagnostic]` on failure.
- [ ] All `FormatDiagnostic*` types are removed.
- [ ] Formatter tests assert shared diagnostic structure or source-aware rendering.
- [ ] `lane fmt` callers compile against the new formatter diagnostic type.
- [ ] `moon info && moon fmt`, `moon check --warn-list +73`, and formatter/lane tests pass.
