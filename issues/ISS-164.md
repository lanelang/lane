## Description
Implement `open-resolve` over the mon-trans output so abstract effect-context opening caused by effect subsumption becomes concrete handler-context adaptation before monadic lifting.

## Design
Compare the callee's expected effect context with the caller's current explicit context arguments. Remove identity openings and lower non-identity openings to ordinary argument selection, projection, reordering, or aggregate construction. Lane performs no global context swap, mask search, or restoration; a resume closure captures the explicit inner arguments it needs. This pass does not create handlers or decide which operation clause runs.

## Acceptance Criteria
- [x] Identical source and target effect contexts remove the abstract opening.
- [x] Empty, singleton, and multiple-effect source contexts adapt explicitly and deterministically.
- [x] Nested handlers of the same effect select the nearest symbolic dictionary binding.
- [x] Generic and polymorphic effect contexts preserve the information required for runtime adaptation.
- [x] Resume closures capture the resolved inner context without a runtime restoration operation.
- [x] No abstract effect-context opening remains in the pass output.

Added `resolve_open_contexts` and an output validator. Mon-trans already materializes context selection, ordering, projection-equivalent references, and context-pack construction as ordinary arguments; open-resolve validates the abstract proof markers, recursively removes them, and preserves those explicit values unchanged. Dedicated tests cover identity, empty, singleton/multiple abstract adaptation, ordered arguments, and residual-marker rejection. All handler semantic fixtures now execute after open-resolve, covering nearest same-effect dictionaries and resume capture without restoration.
