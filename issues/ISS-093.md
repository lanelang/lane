## Description
Add an embedding-facing runtime API that lets MoonBit hosts register handlers for validated runtime effect conventions. Runtime handlers handle only operations that escape all source lexical handlers to the outer runtime boundary.

## Design
Keep Buslane lexical handler semantics unchanged: source handlers remain authoritative, and an unhandled operation without a registered outer runtime handler remains an unhandled operation error. Runtime handler validation should occur before execution at the selected entry boundary and should verify operation payload/result signatures. The handler dispatch should use linked Buslane operation ids internally after validating source-level convention metadata.

## Acceptance Criteria
- [ ] Hosts can register a handler for a validated runtime convention such as `Stdlib.Write.println(String) -> Unit`.
- [ ] Source lexical handlers take precedence over runtime handlers.
- [ ] Runtime handlers only receive operations that reach the outer execution boundary.
- [ ] Invalid handler registration or signature mismatch is reported before execution.
- [ ] Tests cover handled output, source-handler precedence, missing handler, and signature mismatch.
