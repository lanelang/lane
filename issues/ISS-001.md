## Description
`examples/valid/04_blocks_and_local_scope.lane` is a valid example, but running the `factorial` entry currently returns a Buslane verifier/runtime diagnostic instead of a callable function value or successful executable result.

Observed command:

```sh
moon -q run --target native modules/lane -- run examples/valid/04_blocks_and_local_scope.lane --entry factorial --stdlib ../stdlib
```

Observed output:

```text
invalid_program {
  unknown_value value#226
}
```

## Design
Investigate lowering for local recursive functions in block scope. The likely problem is that a local recursive binding references a value id that is not visible in the generated Buslane program. Keep the fix scoped to block/local-function lowering and verifier-facing metadata; do not weaken Buslane verification to accept missing value identities.
