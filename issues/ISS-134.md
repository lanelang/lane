# ISS-134: Replace implicit expected-effect stack with explicit typing context

## Description
`SemanticChecker.expected_effects` is a mutable stack used by operation owner inference. Function and checked-expression code manually call `push_expected_effect` / `pop_expected_effect`, so the correctness of operation inference depends on hidden call ordering and balanced mutation rather than an explicit typing judgment input.

## Design
Introduce an explicit expected typing context, such as an `ExpectedContext` or equivalent, that carries adjacent expected result and expected effect information through checking/synthesis calls. Operation owner inference may consume expected effect evidence only through the generic inference path, but it should not read it from global mutable state.

## Acceptance Criteria
- [x] `SemanticChecker` no longer stores `expected_effects`.
- [x] Operation owner inference still uses expected effect evidence only as generic inference evidence after unique operation selection.
- [x] Function body and operation call tests cover expected-effect-based owner argument inference.
