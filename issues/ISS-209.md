## Description
Replace runtime-import arity-only descriptors with complete direct-value parameter and result signatures supporting primitives plus `Opaque`.

## Design
Serialize ABI major, parameter kind array, result kind, and opaque lookup key. Expand Lane aliases and erase External Types to `Opaque` in LoisVM lowering. Runtime linking validates every kind but intentionally does not preserve External Type fingerprints or payload brands.

## Acceptance Criteria
- [x] Runtime Import descriptors contain complete parameter and result kinds.
- [x] Bytecode encode, decode, disassembly, validation, and tests use the current schema only.
- [x] Lanec lowering derives direct ABI signatures from reachable extern source types.
- [x] Interpreter and Wasm import resolution reject kind mismatches before execution.
- [x] Arity-only descriptor and resolution paths are removed.

## Close Notes
Runtime Import descriptors now encode complete direct ABI signatures with
`Unit`, `Bool`, `Int`, `Double`, `String`, and `Opaque` value kinds. Lanec
derives these signatures from reachable source types, and both execution
backends resolve exact parameter and result kinds before execution. Removed the
old arity-only descriptors, resolver API, and Wasm signature reconstruction.

Verified with `moon check`, `moon test` (806 tests across all targets),
`moon info && moon fmt`, `git diff --check`, and a repository search for the
removed arity-only APIs.
