## Description
Several expressions require an adjacent expected type under Lane's local inference model: unsafe builtins, function literal parameters without annotations, empty matches, and unqualified variant patterns that need an expected enum scrutinee. Today many of these cases report `CannotInferExpression`, which hides the specific missing context.

## Design
Add dedicated typecheck diagnostic variants for each missing expected-type family and render them with targeted titles, labels, notes, and helps. Keep behavior unchanged except for diagnostics.

Non-goals:
- Do not redesign generic argument inference in this issue.
- Do not remove the old catch-all until all dependent migration issues are closed.

## Acceptance Criteria
- [x] Bare `builtin("...")` without expected type reports a builtin-specific diagnostic.
- [x] Function literal parameters without annotations report a parameter-specific diagnostic.
- [x] Empty `match` without expected result type reports an empty-match-specific diagnostic.
- [x] Unqualified variant patterns without expected enum scrutinee type report a variant-pattern-specific diagnostic.
- [x] Existing invalid examples and golden tests are updated to assert the precise diagnostics.
- [x] `moon test modules/lanec/typecheck --warn-list +73 --diagnostic-limit 200` passes.
- [x] `moon test modules/lanec/compile --warn-list +73 --diagnostic-limit 200` passes.
