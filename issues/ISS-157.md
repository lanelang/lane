## Description
The parser result currently represents success with an optional source field plus separate error arrays. That shape hides invalid state combinations and cannot expose concrete syntax sidecars cleanly.

## Design
Change the parser domain result to an enum with parsed and failed states. A parsed result should carry `ParsedSource`, containing the syntax AST and concrete syntax. A parse failure should distinguish lexical failure from grammar failure; grammar failure may carry concrete syntax because lexing succeeded, while lexical failure must not.

## Acceptance Criteria
- [x] `ParseResult` is a domain enum rather than a struct with optional syntax.
- [x] `ParsedSource` contains syntax plus concrete syntax.
- [x] `ParseFailure` distinguishes lexical and grammar failures.
- [x] Existing semantic pipeline callers project syntax without handling formatter trivia.
- [x] Parser diagnostics remain source-aware and existing parser tests are updated.
