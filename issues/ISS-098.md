# ISS-098: Move compiler diagnostic model into lanec diagnostic

## Description
Create `modules/lanec/diagnostic` and move the compiler-facing diagnostic data model out of `modules/lanec/compile`. Rename `CompileDiagnostic*` to `CompilerDiagnostic*` so the model is not owned by the compile orchestration package and can also be used by formatting.

## Design
Add a new MoonBit package `Milky2018/lanec/diagnostic`. Move diagnostic structs and enums into it: `CompilerDiagnostic`, severity, labels, label style, facts, text edits, applicability, and suggestions. Update `lanec/compile` to import the new package and produce the renamed types. Do not add rendering or terminal behavior in this issue.

## Acceptance Criteria
- [ ] `modules/lanec/diagnostic` exists with its own `moon.pkg`.
- [ ] Compiler-facing diagnostic data types live in `modules/lanec/diagnostic`.
- [ ] `CompileDiagnostic*` names are replaced by `CompilerDiagnostic*`.
- [ ] `lanec/compile` produces `CompilerDiagnostic` values from the new package.
- [ ] No terminal, LSP, or source rendering logic is added to `lanec/compile`.
- [ ] `moon info && moon fmt`, `moon check --warn-list +73`, and targeted compiler tests pass.
