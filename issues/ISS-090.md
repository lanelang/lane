# ISS-090: Implement explicit Stdlib runtime effect conventions

## Description
Implement the agreed run/runtime convention for executable Lane entries that use explicit, user-provided `Stdlib` artifacts. The convention must not implicitly inject Stdlib, must not pin an official Stdlib fingerprint, and must validate runtime-handled effects from source-level exported module/effect/operation/signature metadata at the execution boundary.

## Design
Track this as an execution-boundary feature, not a language-level `main` rule. `Stdlib` is a conventional module path whose exported shapes may be supplied by any user artifact. `lane run` and `lane runobj` should execute only zero-argument `Unit` entries whose fully expanded closed concrete effect set is empty or covered by registered runtime conventions. The initial command convention handles only `Stdlib.Write.println(String) -> Unit`.

## Acceptance Criteria
- [ ] `../stdlib` provides the initial `Stdlib` effect and alias shapes.
- [ ] Linked artifacts expose enough source-level exported effect metadata for runtime convention validation.
- [ ] `lanec`/Buslane runtime APIs can validate and invoke outer runtime effect handlers.
- [ ] `lane run` and `lane runobj` execute only valid executable entries and do not debug-render arbitrary public values.
- [ ] ADR-0055 and runtime glossary remain aligned with the implementation.
