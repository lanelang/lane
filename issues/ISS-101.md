# ISS-101: Remove legacy diagnostic pretty paths

## Description
Delete legacy diagnostic pretty/fallback output paths so compiler and formatter diagnostics cannot accidentally bypass source-aware rendering.

## Design
Remove `Pretty for CompileDiagnostic`, `Pretty for FormatDiagnostic`, `CompileDiagnostic::list_doc`, `CliError::FormatDiagnostics`, and all tests that treat compact diagnostic pretty output as user-facing golden output. Update tests to either inspect structured diagnostic values or render through the source-aware adapter. Update `lane fmt` so source diagnostics are printed by the same diagnostic reporting path as check/compile/run. Keep command, artifact, runtime, and Buslane verifier reports separate from compiler source diagnostics.

## Acceptance Criteria
- [x] `Pretty for CompileDiagnostic` is gone.
- [x] `Pretty for FormatDiagnostic` is gone because `FormatDiagnostic` is gone.
- [x] `CompileDiagnostic::list_doc` is gone.
- [x] `CliError::FormatDiagnostics` is gone.
- [x] `lane fmt` uses source-aware diagnostic reporting for formatter diagnostics.
- [x] No CLI or LSP path renders compiler diagnostics through compact pretty fallback output.
- [x] Tests cover source-aware formatter diagnostics and source-aware compiler diagnostics.
- [x] `moon info && moon fmt`, `moon check --warn-list +73`, `moon test`, and `tools/check-lane-run-examples.sh` pass.
