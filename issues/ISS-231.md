## Description
The formatter records concrete trivia but does not assign every comment to one exact printable owner. Span heuristics currently lose, duplicate, or move comments around list items, delimiters, nested expressions, and composite declarations. The previous trivia-preserving epic was closed without an exactly-once ownership guarantee or a real AST-equivalence oracle.

## Design
Keep lexer-owned gap-indexed concrete syntax and the semantic AST free of comments. Replace fuzzy span matching with a formatter-owned attachment model that assigns each comment to one exact token boundary as leading, trailing, dangling, or EOF trivia. Every comment must render exactly once. Extend syntax pretty-printing at list, delimiter, expression, and declaration boundaries rather than adding isolated special cases.

## Acceptance Criteria
- [x] Every supported parse-successful `//` comment is rendered exactly once or formatting fails with E1103 instead of dropping it.
- [x] Comments retain their syntactic attachment across width-sensitive layout changes.
- [x] Module, delimiter, list, expression, declaration, and EOF positions are covered.
- [x] Formatter tests compare syntax before and after formatting and verify idempotence.
- [x] Public formatter APIs cannot silently imply comment preservation from an AST-only input.
- [x] Full in-repository checks, tests, formatting, interface generation, and whitespace checks pass.
- [x] The full examples gate passes against the updated Basic library.
