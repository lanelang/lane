## Description
Design the advanced host-boundary capabilities intentionally excluded from the initial External Effect and External Type implementation.

## Design
Keep these capabilities out of the current direct synchronous ABI until each has an explicit language, ownership, runtime, bytecode, interpreter, and Wasm contract:

- recoverable typed host failures instead of fatal execution panic;
- generic External Types and any required runtime witnesses;
- cross-thread Host Object transfer and concurrency capabilities;
- structured aggregate ABI values such as option, result, arrays, or records.

Do not add ad hoc symbol-specific encodings or weaken the current direct-value ownership rules as an intermediate workaround.

## Acceptance Criteria
- [ ] Each capability has a separately approved ADR before implementation.
- [ ] Bytecode and runtime compatibility consequences are explicit.
- [ ] Interpreter and Wasm ownership and failure semantics remain aligned.
- [ ] Migration from the initial direct ABI is defined where necessary.
