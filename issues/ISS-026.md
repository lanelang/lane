## Description
Buslane text serialization has many `*_text` free helpers. The text format is canonical serialization, not merely debug pretty output, so it needs a deliberate receiver or trait design before a broad rewrite.

## Design
Evaluate whether methods such as `Program::to_text` and `Type::to_text` are possible and appropriate across package boundaries. If package boundaries make those methods awkward, consider a private writer or a small `CanonicalText` trait local to the text package. Do not mix canonical serialization with debug `Pretty` behavior.
