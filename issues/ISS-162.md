## Description
Define the concrete compiler-internal contracts for `mon-trans`, `open-resolve`, and `monadic-lift`. ADR-0067 fixes their order and the effect-free LoisVM boundary but intentionally leaves each pass's intermediate representation and invariants unspecified.

## Design
Choose the minimum compiler-private intermediate forms needed by handler elaboration and the three passes. Specify how deep handlers, operation clauses, final clauses, resume continuations, abstract effect-context opening, latent effects, type parameters, and outer runtime operations are represented after each stage. Decide whether Lane uses explicit immutable handler-context values, another compiler-generated protocol, or a justified runtime mechanism; do not assume Koka's process-global evidence vector fits LoisVM. Keep the design above LoisVM and avoid adding effect concepts to bytecode or VM CFG.

## Acceptance Criteria
- [ ] Handler elaboration and each of the three passes have a documented input grammar and output invariant.
- [ ] The contracts state where deep-handler reinstallation and multi-shot resume semantics are preserved.
- [ ] The contracts state when handler syntax, operation identity, and effect metadata disappear.
- [ ] The contracts define how residual outer runtime operations become ordinary external calls.
- [ ] The contracts identify ownership-relevant allocation and closure boundaries needed by later ARC insertion.
- [ ] ADR-0067 and compiler glossary are updated without changing intentional LoisVM v1 non-goals.
