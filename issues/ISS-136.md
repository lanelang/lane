# ISS-136: Extract operation typing as a deep module

## Description
Effect operation typing currently mixes name selection, owner instantiation, operation-level generic witnesses, handler type binders, checked perform construction, handler arm construction support, and handler coverage substitution in `operation_call.mbt` and `handler.mbt`. `OperationSelection` and `OperationInstantiation` are a step toward locality, but the seam is still too shallow.

## Design
Create a deeper operation typing module/interface that owns operation selection and signature instantiation for both perform sites and handler arms. Perform checking and handler checking should consume an instantiated operation signature instead of partially reproducing owner and witness logic.

## Acceptance Criteria
- [x] Perform operation calls and handler operation arms use the same operation signature instantiation path.
- [x] Handler coverage obtains operation payload types through the same owner-instantiation model.
- [x] Owner elision, operation-level generic witnesses, and handler binders remain covered by regression tests.
- [x] `moon check --warn-list +73`, targeted typecheck tests, and full test suite pass.
