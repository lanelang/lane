## Description
Provide one compiler-private whole-program Core analysis that identifies callable definitions, direct call edges, indirect calls, occurrence facts, expression cost, and recursive call cycles for existing and future Core optimizations.

## Design
Keep the analysis inside `lanec/core_opt`. Reuse the existing occurrence analysis instead of duplicating use and escape facts. Include local callable bindings as graph nodes, classify direct calls through ordinary and type-applied references, and conservatively record unknown calls. Integrate summaries into current generic specialization guards so the infrastructure is exercised by production optimization rather than only white-box tests.

- [x] Existing specialization uses summaries for definition, cost, and recursion decisions.
