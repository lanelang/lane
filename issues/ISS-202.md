# ISS-202: Preserve and erase residual non-monadic effects

## Description
Make monadic translation depend only on open rows and handled operations while preserving non-monadic residual effects such as `Io` until effect-sensitive optimization is complete.

## Design
Use one general `is_mon_effect` predicate. Treat every handled operation as potentially multi-shot and perform no resume-count analysis. Preserve residual `Io` on direct and CPS-transformed callable and continuation types, then run one explicit metadata-only residual-effect-erasure pass after `monadic-lift` and effect-sensitive optimization.

## Acceptance Criteria
- [x] `Io` alone remains direct style while open rows and handled operations trigger monadic translation.
- [x] Mixed `Io` and algebraic-effect programs preserve `Io` through CPS transformation.
- [x] No resume-count or one-shot ABI specialization controls lowering.
- [x] Residual effect erasure removes type-level effects without deleting or reordering extern calls.
- [x] Focused direct, CPS, higher-order, polymorphic, and multi-shot tests pass.
