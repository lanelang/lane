## Description
Implement the schema-v2 image model, codec, formatting, and structural checks for Instance Globals and the optional Instance Initializer.

## Design
Keep schema v1 unchanged. Add dense zero-based GlobalId, global metadata, initializer FunctionId, `init_global`, and `borrow_global` with atomic erased-companion handling as specified by ADR-0114.

## Acceptance Criteria
- [ ] Bytecode image and instruction models expose schema-v2 globals and initializer.
- [ ] Encoder and bitstring-based decoder roundtrip the new records and instructions.
- [ ] Structural and resource-limit failures retain precise diagnostics.
- [ ] Public formatting, info, and codec tests cover v2 while v1 incompatibility is explicit.
