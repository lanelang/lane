# ISS-046: Lower handler payload patterns before Buslane alternatives

## Description
Buslane `OperationAlternative` values bind evaluated payload values
positionally. Lane payload patterns therefore must be eliminated in `lanec`
before any Buslane operation alternatives are created.

## Design
Introduce a pattern-free checked or lowering-ready handler representation that
keeps operation alternatives grouped by handled effect but replaces source
payload patterns with value binders and explicit match logic in Lane lowering.
Keep source spans and source pattern presentation outside Buslane nodes.

## Acceptance Criteria
- [ ] Handler lowering input has no source payload patterns at the point where
  Buslane `OperationAlternative` values would be built.
- [ ] Pattern lowering preserves the behavior of direct payload binders,
  constructor patterns, tuple patterns, and wildcard patterns supported by the
  existing Lane pattern machinery.
- [ ] Source spans and pattern presentation remain available for diagnostics
  without being stored in Buslane nodes.
- [ ] Tests or snapshots show the lowered handler shape for direct arms and
  pattern arms.
- [ ] `moon check modules/lanec --warn-list +73` passes.
- [ ] `moon info && moon fmt` has been run after the change.
