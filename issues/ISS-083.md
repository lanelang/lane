## Description
Add completion candidates for nominal/effect owner `::` contexts and checked receiver selector `expr.` contexts.

## Design
For `Type::` and `Lib.Type::`, suggest owner-defined variants or constructor-like members according to the source language. For `Effect::` and `Lib.Effect::`, suggest operations, including operation owners from imported module interfaces. For `expr.`, perform selector completion only: Lane has no method concept, so suggest struct fields and any existing type-member access forms supported by the language, but do not search ordinary functions by receiver type. Receiver selector completion requires a checked receiver type; return no typed selector candidates when the type is unavailable.

## Acceptance Criteria
- [x] Nominal owner completion suggests variants/constructors for local and imported owners.
- [x] Effect owner completion suggests operations for local and imported effects.
- [x] `Lib::x` is not treated as module-member completion.
- [x] Checked selector completion suggests fields/type members for known receiver types.
- [x] Receiver selector completion returns empty results when typecheck data is unavailable.
