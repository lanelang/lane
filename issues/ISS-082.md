## Description
Collect semantic completion candidates for ordinary identifier contexts, module-qualified contexts, and import declaration module paths.

## Design
Reuse resolver/module input knowledge rather than scanning text in the LSP layer. Ordinary identifier completion should use visible lexical scope and expected namespace to suggest values, functions, types, type parameters, effects, effect aliases, offers, and modules as appropriate. Module-qualified completion should expose only the imported module interface surface. Import completion should suggest module paths already present in the module input set or workspace inputs and must not add module-name filesystem search.

## Acceptance Criteria
- [x] Ordinary identifier completion suggests visible local and top-level candidates with stable sorting.
- [x] Type and effect positions filter out inappropriate value-only candidates.
- [x] Module-qualified completion exposes exported module surface only.
- [x] Import declaration completion suggests known module paths and excludes the root module.
- [x] Typecheck failures do not suppress resolve-level lexical/module/import completion when resolve data is available.
