# ISS-151: Remove field offer from the Lane surface model

## Metadata
- Type: task
- Status: closed

- Assignee: unassigned

## Description
Struct fields currently support an `offer` modifier, making a field become a contextual offer when the containing value is offered. This makes contextual offers non-obvious and can create ambiguity from derived field paths. Remove this surface feature entirely rather than keeping an ignored compatibility path.

## Design
Delete field-level offer syntax and the corresponding field-model bits from syntax, resolve, checked, module interface, artifacts, pretty printers, inspection, and completion-facing surfaces. Keep value parameters unchanged: `auto`, `offer`, and `auto offer` remain valid and orthogonal.

## Acceptance Criteria
- [x] Parser rejects `struct S { offer field : T }`.
- [x] Field models no longer carry `is_offered`.
- [x] Public module interfaces and artifact binary payloads no longer encode field-level offer state.
- [x] Generated parser and `.mbti` files are updated.
