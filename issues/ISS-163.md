## Description
Implement `mon-trans` over the handler-elaborated representation established by ISS-162 and ISS-168. The pass must make the continuation after calls that can resume multiple times or run under an open effect context explicit without changing observable Lane semantics.

## Design
Use continuation builders or an equivalent selective transformation to insert explicit bind points around potentially resumptive calls while leaving total direct-style work direct. Preserve strict left-to-right evaluation and share one continuation after effectful branches. Keep abstract handler-context opening intact for `open-resolve`. Do not emit ANF, VM CFG, ARC instructions, or LoisVM bytecode in this pass.

## Acceptance Criteria
- [ ] Calls that can multi-resume or have an open effect context receive explicit continuation binds.
- [ ] Calls proven direct retain ordinary direct-style evaluation.
- [ ] Function and argument evaluation order remains left to right.
- [ ] Effectful match branches join through one shared continuation.
- [ ] Identity continuations do not introduce redundant binds.
- [ ] Type, effect-context opening, and source identity required by the next pass are preserved.
