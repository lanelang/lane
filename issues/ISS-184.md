## Description
Ensure each monadic function's generated answer-type binder is identical in its transformed `TypeLambda` expression and its rewritten value metadata.

## Design
Selective CPS must transform all terms while source metadata still describes source latent effects. After the source metadata rewrite, resynthesize each transformed top-level binding and overwrite its value type with that exact result. External values keep their independently rewritten callable type because they have no defining type abstraction.

## Acceptance Criteria
- [x] A transformed monadic function uses the same `TypeParameterId` in its term binder and value metadata.
- [x] Recursive and non-recursive top-level function bindings are synchronized after all terms are transformed.
- [x] Cross-function monadic calls still classify from source metadata during transformation.
- [x] LoisVM lowering receives the generated Answer layout witness in scope.
- [x] Effect-lowering tests and the targeted production Answer-witness boundary pass without warnings.

## Notes
- 2026-07-13: Production integration exposed a body binder at type-parameter index 1 while metadata independently named index 2, causing `missing in-scope layout witness`.

## Close Notes
After all source terms are transformed and source metadata is rewritten, mon-trans now resynthesizes every transformed top-level `Let`/`LetRec` binding and installs that exact type into value metadata. This preserves source-effect classification during transformation while making each generated `TypeLambda` Answer binder identical to the metadata `Forall` binder consumed by LoisVM layout-witness lowering. A regression asserts binder identity directly; the production runtime-effect test now advances beyond the previous missing-witness failure into VM CFG ownership validation tracked separately.
