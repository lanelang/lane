## Description
Name resolution diagnostics are currently rendered by a large `resolve_diagnostic` switch in `lanec/compile`. That makes compile orchestration own wording for unresolved values, missing fields, duplicate imports, ambiguous variants, and similar resolve facts.

## Design
Move the rendering of `ResolveDiagnostic` into the resolve package or a resolve-owned diagnostics file. Compile should only call the typed conversion and attach source ids.

## Acceptance Criteria
- [x] `ResolveDiagnostic` implements or delegates to the shared typed diagnostic API.
- [x] `lanec/compile` no longer contains the resolve diagnostic wording switch.
- [x] Missing value, missing field, duplicate top-level value, ambiguous import, and variant resolution golden tests still pass.
- [x] LSP diagnostics still carry source ranges and data correctly.
- [x] `moon test modules/lanec/compile --warn-list +73` passes.
