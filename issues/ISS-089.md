# ISS-089: Insert parentheses for payloadless variant completions

## Description
Semantic completion for owner-qualified enum variants currently inserts the bare variant name. For payloadless variants, this produces invalid Lane source under the current constructor syntax rule. For example, completing `Option::` can produce `none` instead of `none()`.

## Design
Use the variant payload metadata already present in checked declarations, resolved declarations, and module interfaces. For payloadless variants, set `insert_text` to `name + "()"` while keeping the visible label as the variant name. For payload-bearing variants, keep the existing insertion behavior unless a separate snippet policy is designed later.

## Acceptance Criteria
- [x] Compiler completion entries for local, resolved-fallback, and imported payloadless variants insert `name()`.
- [x] LSP completion `textEdit.newText` for payloadless variants is `name()`.
- [x] Tests cover `Option::none()` insertion while preserving `some` label behavior.
- [x] `moon test modules/lanec/compile` and `moon test modules/lane/lsp` pass with `--warn-list +73`.
