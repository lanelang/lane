# ISS-121: Preserve generic inference failure details

## Description
Generic argument inference currently collapses several outcomes into `None` and then reports `CannotInferExpression(..., GenericArguments)`. The diagnostic cannot tell whether an argument was missing, a collected constraint conflicted, or a kind/type shape made inference impossible.

## Design
Return a structured generic inference failure value from the generic argument inference layer. Include missing parameter identifiers where possible and preserve constraint mismatch facts without introducing global constraint solving.

## Acceptance Criteria
- [x] Generic inference returns structured failure data instead of a bare `None` for reportable failures.
- [x] Missing generic arguments identify the callee and the generic parameter(s) that could not be inferred.
- [x] Constraint mismatch failures distinguish conflict from missing context.
- [x] Diagnostics no longer use `CannotInferExpression(..., GenericArguments)` for migrated generic inference paths.
- [x] Focused typecheck and compile golden tests cover missing and conflicting generic argument inference.
