## Description
Contextual offer lookup and call-shape validation still report some failures through the broad inference failure path. Missing or ambiguous contextual offers, duplicate named arguments, invalid auto argument placement, and invalid named builtin arguments should not be presented as generic type inference problems.

## Design
Add dedicated diagnostics for contextual offer lookup and call-shape failures. Keep call checking behavior unchanged while giving users concrete labels and fixes.

## Acceptance Criteria
- [x] Missing contextual offers identify the required contextual type.
- [x] Ambiguous contextual offers list or point to the relevant candidates.
- [x] Invalid auto argument and duplicate named argument failures are not reported as type inference failures.
- [x] Invalid named arguments to builtin calls get call-shape diagnostics.
- [x] CLI/LSP-visible diagnostic fixtures are updated where outputs change.
