## Description
`modules/lanec/typecheck/typecheck_wbtest.mbt` still contains a test where `let value = none` expects a typecheck diagnostic about unresolved variant type arguments. Under the current syntax rule, bare payloadless variant constructors should be rejected as missing empty parentheses before that internal typecheck scenario is treated as the user-facing error.

## Design
Adjust the test so it does not lock an invalid source program as a typecheck inference case. Either use a payload-bearing generic variant expression that genuinely reaches typecheck inference, or add a helper/test path that explicitly accounts for resolver diagnostics before typechecking. Do not let typecheck whitebox tests silently ignore resolver diagnostics for source snippets meant to represent valid resolver output.

- [x] A replacement test still covers the intended unresolved type-argument behavior with resolver-valid source.
