## Description
Lane LSP currently supports diagnostics-first document synchronization, but it does not support `textDocument/definition` or `textDocument/hover`. Add semantic editor support by having `lanec` produce a queryable analysis result and having `lane lsp` adapt that result to LSP requests.

## Design
Keep semantic ownership in `lanec`; `lane lsp` should remain a protocol and document-overlay adapter. Do not implement navigation by string matching in the LSP server. The analysis path must preserve source identity for the root and library inputs so cross-module locations can be returned with URIs.

## Acceptance Criteria
- [ ] `lane lsp` advertises definition and hover capabilities only after the backing semantic query path exists.
- [ ] Definition lookup works for at least local values, top-level values, nominal types, fields, and enum variants.
- [ ] Hover returns stable type, kind, or declaration summaries for supported semantic nodes.
- [ ] LSP tests cover successful same-file and cross-module navigation or document the unsupported subset.
- [ ] `moon test --target native --warn-list +73` passes.
