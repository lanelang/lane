## Description
Introduce `Io` as an import-free built-in effect atom throughout source typing and typed compiler IR. It must have intrinsic identity, participate in effect rows and polymorphism, affect purity, and remain distinct from nominal algebraic effects.

## Design
Represent `Io` explicitly rather than reserving or synthesizing an effect identity. Extend resolution, checked types, Buslane effects, substitution, canonicalization, printing, interfaces, and their binary and text codecs. Reject using `Io` as a handleable algebraic effect. Monadic classification and final artifact schema migration remain owned by ISS-202 and ISS-203 respectively.

## Acceptance Criteria
- [x] Lane accepts `Io` in effect rows without an import.
- [x] `Io` roundtrips through checked and Buslane type representations and their codecs.
- [x] Effect canonicalization and polymorphic substitution preserve `Io` correctly.
- [x] `Io` remains distinct from `Empty` and cannot be handled or resolved as a nominal effect.
- [x] Focused type, effect, interface, and artifact tests pass.
