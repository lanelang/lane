## Description
Lane already has the major v1 path from resolved source through checked source into Buslane, and successful compilation can lower source-only syntax such as operators, pipelines, field access, builtins, handlers, effects, and complex patterns before Buslane. However, the long-term boundary is still implicit and spread across resolver, desugar, typecheck, checked source, Buslane lowering, and roadmap notes.

Define a hard pre-Buslane contract for elaboration, canonicalization, and source lowering. The contract should state exactly what a checked source success value guarantees before Buslane lowering begins, and tests should prevent source-only syntax, unresolved or ambiguous state, alias leaks, missing contextual arguments, or missing generic/effect witnesses from crossing the Buslane boundary.

## Design
Treat this as a boundary-hardening issue, not a performance optimization issue.

The intended shape:
- Checked Source success has explicit invariants: no unresolved names, no omitted contextual arguments, no source-only ambiguity states, and all source-only syntax either desugared or represented in a checked form with a required lowering rule.
- Type/effect canonicalization has a single documented ownership model: transparent aliases expand before Buslane, definitional type equality and beta normalization are applied in the type layer, and source type presentation remains available only for diagnostics.
- Buslane lowering accepts only checked semantic data and produces no source spans, source names, parser nodes, resolver nodes, checked AST nodes, field nodes, `if` nodes, unsafe-builtin nodes, or source pattern nodes.
- Origin and presentation metadata required by diagnostics or LSP lives outside Buslane in explicit side data.
- Roadmap and context docs reflect the current F-omega, generic argument, alias, effect, and Buslane lowering state instead of stale checklist entries.

Non-goals:
- Do not add Core or runtime performance optimizations here.
- Do not introduce source spans or diagnostic presentation into Buslane nodes.
- Do not split this epic into multiple issues unless implementation discovers an independently schedulable blocker.

## Acceptance Criteria
- [x] A documented pre-Buslane contract defines the required checked-source invariants and the forbidden Buslane-boundary leaks.
- [x] Checked Source construction or an adjacent validation pass rejects or reports unresolved names, omitted contextual arguments, source-only ambiguity states, and source-only forms that lack a lowering rule.
- [x] Type and effect canonicalization ownership is documented and covered by tests for transparent aliases, effect aliases, higher-kinded applications, type-level lambda normalization, and generic/effect arguments.
- [x] Buslane lowering tests assert that representative source forms lower without leaking source syntax or losing witnesses: operators, pipelines, field access, builtins, if, matches, pattern lets, handlers, operation calls, constructor calls, existential witnesses, and generic operation witnesses.
- [x] Diagnostic and LSP origin/presentation data needed after lowering is kept outside Buslane and has regression coverage.
- [x] `modules/lanec/docs/roadmap-checklist.md`, `modules/buslane/docs/roadmap-checklist.md`, and context docs are updated to match the actual boundary.
- [x] `moon check --warn-list +73`, `moon test --warn-list +73`, and `tools/check-lane-run-examples.sh` pass after the contract is enforced.
