## Description
Lane type checking currently computes only value types. Effect operation calls
and function latent effects require checked expressions to carry effect sets
alongside value types.

## Design
Extend the Lane type model with effect kinding, effect terms, function latent
effects, and expression judgments that synthesize both value type and effect
set. Add source effect annotations for function types and named function
declarations. Operation calls should resolve to operation identities and check
payload arguments before producing a checked operation-call expression.

## Acceptance Criteria
- [x] `Kind::Effect` and source effect terms are represented in `lanec` type
  data.
- [x] Function types and function declarations carry latent effect
  annotations; omitted annotations remain pure.
- [x] Type checker synthesizes expression effects and rejects non-empty effects
  where pure expressions are required.
- [x] Effect operation calls check payload arity/types and produce checked
  operation-call nodes.
- [x] Effect row variables are represented as kind-`Effect` type parameters.
- [x] Typechecker tests cover pure functions, annotated effectful functions,
  operation calls, and effect mismatch diagnostics.
- [x] `moon test modules/lanec` passes.
- [x] `moon info && moon fmt` has been run after the change.
