## Description
`lane compile`, `lane link`, `lane exec`, and `lane inspect` should read and write real binary `.lmi`, `.lmo`, and `.lbp` artifacts. The current binary wrapper around UTF-8 artifact text should be replaced by structured binary payload schemas, and artifact text parsing should be removed from the production artifact load path.

## Design
Rewrite `modules/lanec/compile/artifact_binary.mbt` and related artifact IO to use:

- Lane artifact container version for magic, kind, payload length, and future framing features.
- artifact schema versions for interface, module object, and linked program payload field order.
- `bytecodec` for primitive fields.
- `buslane/codec` for Buslane/core sections.
- typed artifact decode `suberror`s caught at command/tool boundaries and rendered as artifact load diagnostics.

Remove official dependence on `modules/lanec/compile/artifact_parser` and `artifact_text` for load paths. Keep `inspect` as semantic output over decoded binary artifacts and Buslane pretty output. Do not add a legacy compatibility reader for old text-payload artifacts.

## Acceptance Criteria
- [x] `.lmi`, `.lmo`, and `.lbp` are written with structured binary payloads, not UTF-8 artifact text.
- [x] `lane compile -i`, `lane link`, `lane exec`, and `lane inspect` decode structured binary payloads.
- [x] Production code no longer calls artifact text parsers for artifact loading.
- [x] Artifact text parser tests are removed or replaced with binary roundtrip, malformed binary, CLI smoke, and inspect golden tests.
- [x] Existing artifact inspect output remains human-readable and includes Buslane pretty code for object/program artifacts.
- [x] `moon info && moon fmt`, `moon test`, and relevant CLI smoke tests pass.
