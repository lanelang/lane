## Description
Move compiler diagnostic conversion into `modules/lanec/diagnostic` so CLI and LSP presentation layers do not duplicate severity, label, fact, suggestion, and source-id mapping logic.

## Design
Add `CompilerDiagnosticSources` to `modules/lanec/diagnostic`. It wraps diagnostic source infrastructure and maps Lane compiler `source_id` values to generic diagnostic source ids. Add conversion from `CompilerDiagnostic` to `diagnostic/core.Diagnostic`. Update `modules/lane/reporting.mbt` and `modules/lane/lsp/diagnostics.mbt` to call this adapter and keep only terminal/LSP presentation logic. `CompilerDiagnosticSources` must accept source inputs/text supplied by callers; it must not read files, discover files, or know LSP URIs.

## Acceptance Criteria
- [x] `modules/lanec/diagnostic` exposes `CompilerDiagnosticSources`.
- [x] `CompilerDiagnostic` converts to `diagnostic/core.Diagnostic` in one package.
- [x] `lane/reporting.mbt` no longer implements compiler diagnostic conversion.
- [x] `lane/lsp/diagnostics.mbt` no longer implements compiler diagnostic conversion.
- [x] LSP diagnostics still provide protocol-required ranges, including fallback behavior for source-less or unmapped diagnostics.
- [x] CLI source diagnostics still render with source snippets where source text is available.
- [x] `moon info && moon fmt`, `moon check --warn-list +73`, LSP tests, and lane tests pass.
