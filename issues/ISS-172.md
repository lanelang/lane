## Description
Use typed effect-core synthesis to make continuation boundaries explicit around calls and branches that have nonempty or abstract effects while preserving direct style for pure computations.

## Design
Extend compiler-private effect core with typed `Bind` and abstract `OpenContext` plans. Transform expressions through continuation builders in strict left-to-right order, share one continuation after effectful matches, and remove identity continuations. Preserve `Invoke` and `Install` temporarily for ISS-173 to lower with the selected continuation and context.

## Acceptance Criteria
- [x] Nonempty and abstract-effect calls receive explicit bind continuations.
- [x] Pure calls and pure sequencing remain direct style.
- [x] Callee and argument evaluation remains left to right.
- [x] Effectful match branches share one continuation.
- [x] Identity continuations do not create redundant binds.
- [x] Structural tests cover nested calls, lets, matches, and effect-polymorphic calls.
