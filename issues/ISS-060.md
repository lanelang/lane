# ISS-060: Add effect and operation symbols to LSP analysis

## Description
Lane effect declarations are checked by the compiler, but checked analysis does
not publish effect or operation declarations as definition symbols. As a result,
the LSP cannot provide hover or goto-definition on effect and operation
declaration names even though these are first-class source entities.

## Design
Extend the analysis symbol model with effect and operation symbols, then collect
definitions and declaration hovers from checked effect declarations. Keep this in
the compiler analysis layer so the LSP remains a transport over analysis data.

## Acceptance Criteria
- [x] `AnalysisSymbol` can represent effect and operation symbols.
- [x] Checked effect declarations emit definition and hover entries for the effect name.
- [x] Checked effect operation declarations emit definition and hover entries for the operation name.
- [x] Effect and operation type parameter binders keep their existing type-parameter editor behavior.
- [x] LSP tests cover effect and operation declaration hovers or definitions.
