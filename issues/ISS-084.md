## Description
Wire compiler semantic completion into `lane lsp` as `textDocument/completion`.

## Design
Advertise `completionProvider` with trigger characters `.` and `:` and no resolve provider. Parse completion params, map manual and trigger-character requests to compiler `CompletionTrigger`, and return a LSP `CompletionList` with `isIncomplete=false`. Map compiler completion kinds, detail, documentation, sort text, and replacement range into LSP completion items. Do not generate snippets or resolve-provider follow-up requests in v1.

## Acceptance Criteria
- [x] `initialize` advertises a completion provider with `.` and `:` trigger characters.
- [x] `textDocument/completion` validates params and returns JSON-RPC invalid params errors for malformed requests.
- [x] Completion responses use `CompletionList` with `isIncomplete=false`.
- [x] LSP items include correct labels, kinds, details, sort text, and text edit ranges.
- [x] LSP tests cover manual requests, `.` trigger routing, and `:` trigger routing for `::`.
