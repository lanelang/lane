# ISS-199: Separate extern syntax from compiler builtins

## Description
Add `extern("...")` as the open host-linkage source form and redefine `builtin("...")` as a closed compiler-intrinsic request with compiler-owned contracts.

## Design
Give builtin and extern expressions distinct syntax, resolved, and checked origins. Extern checking requires a complete expected monomorphic function type over the primitive host ABI with latent effect `Empty` or exactly `Io`. Unknown builtins, builtin signature mismatches, incomplete extern expectations, and excluded extern shapes are compile errors.

## Acceptance Criteria
- [ ] `extern("name")` parses, formats, resolves, and appears distinctly in checked IR.
- [ ] Externs require complete expected function types and reject scalar, generic, aggregate, open-row, and algebraic-effect signatures.
- [ ] Extern latent effects are accepted only when empty or exactly `Io`.
- [ ] Builtins are validated against a closed compiler-owned name and signature table with no runtime fallback.
- [ ] Complete structured diagnostics are covered by black-box or appropriate package tests.
