## Description
The LSP test suite covers values, types, fields, variants, and type parameters,
but it has no blackbox coverage for effect declarations, operation calls, or
handler operation arms. This lets effect editor regressions pass unnoticed.

## Design
Add focused LSP tests around definition and hover requests for effect and
operation syntax. Keep the tests at the LSP boundary so they validate the full
analysis-to-transport path.

## Acceptance Criteria
- [x] LSP tests include effect declaration hover or definition behavior.
- [x] LSP tests include polymorphic operation call definition or hover behavior.
- [x] LSP tests include handler operation arm definition or hover behavior.
- [x] `moon test modules/lane/lsp` passes.
