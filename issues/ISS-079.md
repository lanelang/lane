## Description
Add Lane semantic completion for editor tooling. Completion should be owned by `lanec` as a position-specific compiler analysis query and transported by `lane lsp` as LSP completion items.

## Design
Follow ADR-0057. V1 completion covers semantic candidates only: visible lexical names, module-qualified members, nominal/effect `::` members, checked selector fields/type members, and import module paths. Do not add keyword snippets, method search, auto-imports, or editor-side text scanning. LSP should be a protocol adapter over compiler-owned completion entries.

## Acceptance Criteria
- [x] `lanec` exposes a position-specific completion query API.
- [x] Completion returns semantic entries with replacement ranges, kinds, details, and stable sorting.
- [x] `lane lsp` advertises and serves `textDocument/completion`.
- [x] Compiler and LSP tests cover semantic selection and transport mapping.
