## Description
Implement the accepted host-boundary design for user-declared External Effects and zero-parameter External Types, including their erased `Opaque` runtime ABI, owned host-object lifecycle, interpreter behavior, Wasm behavior, and typed host registration surface.

## Design
Use `extern type E : Effect` for non-handleable observable host dependencies and `extern type T : Type` for opaque nominal host objects. Preserve their semantic flavors through checking, Buslane, and linking, then erase External Types to one direct `Opaque` ABI kind during LoisVM lowering. Keep host objects in an execution-local generational table with Lane-owned ARC wrappers, borrowed parameters, owned results, and per-instance finalizers.
