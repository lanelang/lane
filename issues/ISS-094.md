# ISS-094: Make run and runobj execute only executable entries

## Metadata
- Type: feature
- Status: open
- Priority: 1
- Labels: area/cli, area/runtime, area/effects, agent
- Assignee: unassigned
- Created: 2026-07-04
- Updated: 2026-07-04
- External ref: none

## Description
Change `lane run` and `lane runobj` from debug-rendering arbitrary public entries to executing only executable entries. An executable entry is a zero-argument function returning `Unit` whose fully expanded closed concrete effect set is empty or covered by registered runtime conventions.

## Design
Validate selected entries at the execution boundary. Accept pure `() -> Unit` and entries whose expanded concrete effects are covered by runtime conventions such as `Stdlib.Write`; reject extra effects, effect row variables, non-`Unit` results, value entries, and functions with parameters. The default command runtime should register only `Stdlib.Write.println(String) -> Unit`, and should not print the `Unit` result.

## Acceptance Criteria
- [ ] `lane run` and `lane runobj` execute pure `() -> Unit` entries successfully without printing `Unit`.
- [ ] `lane run` and `lane runobj` execute `() -> Unit ! Stdlib.Write` entries and print output through `Stdlib.Write.println`.
- [ ] `run/runobj` reject arbitrary public value entries and non-executable function types with clear diagnostics.
- [ ] Entry validation uses expanded closed concrete effect sets, so aliases like `Stdlib.Io`, `Stdlib.Console`, and `Stdlib.Write` behave equivalently when they expand to the same set.
- [ ] Tests cover explicit Stdlib compile/link inputs and confirm no implicit Stdlib injection.
