## Description
Introduce the small shared API that compiler phases will implement when converting typed diagnostic facts into `CompilerDiagnostic`. This is the MoonBit replacement for rustc's derive-based `Diagnostic` implementations.

## Design
Add a `DiagnosticContext` value and a `ToCompilerDiagnostic` trait in `modules/lanec/diagnostic`. Also add low-level builder helpers for common diagnostic shapes: `compiler_diagnostic`, `primary_label`, `compiler_warning`, and optional source-id application. The API must stay domain-neutral: no parser/typecheck/resolve-specific wording or imports.

This issue should not migrate large diagnostic families yet. It only creates the stable target shape for later owner-local implementations.

## Acceptance Criteria
- [x] `modules/lanec/diagnostic` exports a domain-neutral `DiagnosticContext`.
- [x] `modules/lanec/diagnostic` exports a `ToCompilerDiagnostic` trait.
- [x] Shared builder helpers can construct `CompilerDiagnostic` without depending on `lanec/compile`.
- [x] Existing diagnostics continue to compile and render unchanged.
- [x] `moon info && moon fmt` and `moon check --warn-list +73` pass.
