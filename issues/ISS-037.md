## Description
After Buslane can represent and verify the effect core, `lanec` must lower
Lane effect declarations, operation calls, and handlers into Buslane without
leaking source syntax into Buslane nodes.

## Design
Extend the checked-source or lowering input data with effect declarations and
operation signatures once the source layer is ready. Lower effect metadata,
operation calls as `perform`, and handlers as Buslane handler tables grouped by
effect. Lower source payload patterns before Buslane so operation alternatives
only bind positional payload values. Preserve source presentation and spans in
diagnostic-side data outside Buslane.

## Acceptance Criteria
- [ ] Lane effect declarations lower to Buslane effect and operation metadata.
- [ ] Lane operation calls lower to Buslane `perform`.
- [ ] Lane handlers lower to Buslane handler tables grouped by effect.
- [ ] Source payload patterns are eliminated before Buslane operation
  alternatives.
- [ ] Source spans and presentation remain outside Buslane nodes.
- [ ] Integration snapshots cover Lane source through checked source into
  Buslane effect core.
- [ ] `moon test modules/lanec` passes.
- [ ] `moon info && moon fmt` has been run after the change.
