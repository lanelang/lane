## Description
Elaborate Buslane `Handle`, `Perform`, and `Resume` syntax into the compiler-private handler and continuation protocol selected by ISS-162 before the three monadic passes run. Unlike Koka Core, current Buslane still carries explicit handler nodes at this boundary.

## Design
Translate handler tables, operation clauses, final clauses, and resume binders into explicit compiler-generated values and operations while preserving nearest-handler dispatch, deep-handler reinstallation, and multi-shot resume. Introduce abstract effect-context opening where effect subsumption requires a callee-specific context. Keep the representation compiler-private and independent of LoisVM bytecode.

## Acceptance Criteria
- [x] Buslane `Handle`, `Perform`, and `Resume` are absent from the elaborated output.
- [x] Handler installation and operation clauses use the protocol defined by ISS-162.
- [x] Resume binders become ordinary references ready for mon-trans to supply deep continuation closures.
- [x] Each final clause is represented by exactly one typed ordinary function.
- [x] Effect and type metadata required for later abstract context opening remain attached to invokes, dictionaries, and callable types.
- [x] Exact structural tests cover direct invocation, final clauses, generic operation clauses, and multi-shot resume references.
