# ISS-153: Migrate Basic operations away from field offer

## Metadata
- Type: task
- Status: closed

- Assignee: unassigned
- Created: 2026-07-08
- Updated: 2026-07-08
- External ref: none

## Description
Basic currently uses `Compare[T]` with an offered `equal_impl : Equal[T]` field. With field offer removed, Basic must explicitly offer equality and comparison operation values where both are intended to be available.

## Design
Make `Compare[T].equal_impl` a normal field. Publish explicit `Equal` offers for primitive types alongside explicit `Compare` offers. Update examples and fixtures so code that directly uses `==` has a visible `Equal` offer, and code that directly uses `<` has a visible `Compare` offer.

## Acceptance Criteria
- [x] Basic exposes explicit `Equal[Int]`, `Equal[Double]`, `Equal[Bool]`, and `Equal[String]` offers as needed.
- [x] Basic exposes explicit `Compare[Int]` and `Compare[Double]` offers that use the corresponding equal ops internally.
- [x] Valid examples using equality and comparison still pass.
- [x] Invalid examples cover missing equality when only comparison is offered.
