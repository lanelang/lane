# ISS-234: Preserve comments inside expressions and declarations

## Description
Comments after operators, before initializer expressions, around match arrows and handler syntax, and after closing braces of functions or nominal declarations can be lost or moved because only selected whole-node spans consume trivia.

## Design
Expose trivia boundaries for expression operators, assignment separators, arrows, branch keywords, and composite declaration ends. Preserve syntactic ownership while allowing the pretty-printer to choose canonical line breaks.

## Acceptance Criteria
- [x] Unary, binary, pipeline, call, field, `if`, `match`, and handler comment positions survive.
- [x] Assignment and initializer comments remain on the initializer boundary.
- [x] Function, struct, enum, effect, type alias, and local declaration trailing comments survive.
- [x] Corrected outputs parse to equivalent syntax and are idempotent.
