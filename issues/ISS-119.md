# ISS-119: Replace catch-all inference diagnostics with structured cases

## Description
Type inference failures currently flow through `CannotInferExpression(label, reason)`. That catch-all preserves only a string label and a coarse reason, so very different failures are rendered as the same vague "cannot infer expression type" diagnostic.

This epic tracks the long-term fix: typechecking must emit structured semantic diagnostics for each inference failure family, while compile diagnostic rendering turns those facts into precise source messages.

## Design
Keep Lane's local/bidirectional inference model. Do not introduce global constraint solving. Replace the catch-all with specific diagnostic kinds such as missing expected type, generic argument inference failure, contextual offer failure, and invalid call shape.

The final state should remove `CannotInferExpression`, `CannotInferReason`, `cannot_infer_diagnostic`, and `report_application_inference_failure` from normal user-facing paths.

## Acceptance Criteria
- [x] User-facing inference failures no longer render as a generic catch-all for known cases.
- [x] Typecheck diagnostics model missing context, generic inference failure, contextual offer failure, and call-shape errors separately.
- [x] Diagnostic rendering explains what information is missing, why local inference could not obtain it, and where the user can add it.
- [x] Golden and examples fixtures cover the major inference-failure families.
- [x] The obsolete catch-all diagnostic and helper are deleted or restricted to an internal impossible-state path.
