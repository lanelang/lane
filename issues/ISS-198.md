## Description
Introduce `Io` as an import-free built-in effect atom throughout source typing and typed compiler IR. It must have intrinsic identity, participate in effect rows and polymorphism, affect purity, and remain distinct from nominal algebraic effects.

## Design
Represent `Io` explicitly rather than reserving or synthesizing an effect identity. Extend parsing, resolution, checked types, Buslane effects, substitution, canonicalization, printing, interfaces, and typed artifact paths as required. Reject using `Io` as a handleable algebraic effect and ensure `Io` alone does not imply monadic translation.

## Acceptance Criteria
- [ ] Lane accepts `Io` in effect rows without an import.
- [ ] `Io` roundtrips through checked and Buslane type representations and typed artifacts.
- [ ] Effect canonicalization and polymorphic substitution preserve `Io` correctly.
- [ ] `Io` contributes to impurity but cannot be handled or resolved as a nominal effect.
- [ ] Focused type, effect, interface, and artifact tests pass.
