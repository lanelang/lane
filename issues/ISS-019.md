## Description
The command-line and LSP layers still expose terse operational errors. Some are user mistakes, such as a malformed run target; others are protocol errors or internal compiler/runtime failures. They should be classified clearly so users know whether to edit their program, adjust CLI inputs, recompile artifacts, or file a compiler bug.

Current message inventory:

- `unknown lane command`
- `missing argument: ...`
- `run target must be FILE:ENTRY`
- `failed to read source: ...`
- `failed to read library source: ...`
- `compile requires --emit-interface`
- `compile requires --emit-object`
- `failed to read module interface artifact: ...`
- `failed to write module interface/object/linked program artifact: ...`
- `entry not found: ...`
- `ambiguous entry: ...`
- `invalid didOpen/didChange/didClose params`
- `invalid definition params`
- `invalid hover params`
- `invalid semantic tokens params`
- `method not found: ...`
- runtime/interpreter strings such as `if condition is not Bool`, `callee is not callable`, `argument count mismatch`, `field base is not a struct`, `missing field ...`

## Design
Separate user-facing recoverable errors from internal compiler bugs. CLI command errors should include command usage hints and examples. LSP protocol errors can stay JSON-RPC shaped, but their messages should include the missing/invalid field. Runtime boundary errors should either be unreachable after typechecking and reported as internal compiler bugs, or intentionally documented runtime reports with actionable context.

## Acceptance Criteria
- [ ] CLI argument errors include the expected form and one valid example.
- [ ] File IO errors include the path being read/written and preserve host error text.
- [ ] Entry lookup errors list available entries when practical.
- [ ] LSP invalid-params errors identify the bad request field and do not blame Lane source code.
- [ ] Runtime errors are classified as user runtime reports or internal compiler bugs, with distinct wording.
- [ ] Tests cover malformed run target, missing compile outputs, missing entry, LSP invalid params, and at least one runtime boundary report.
