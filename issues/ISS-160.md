## Description
Snapshot tests alone are not enough for a trivia-preserving formatter. Add structural verification that proves comments are not lost, formatting does not change the AST, and formatting is idempotent.

## Design
Create formatter verification helpers for comment no-loss, AST roundtrip equivalence, and idempotence. Add fixtures covering comments around declarations, imports, blocks, trailing comments, handler syntax, lists, field access, pipelines, if/else, match arms, struct/enum/effect declarations, and blank-line grouping.

## Acceptance Criteria
- [x] Formatter tests fail if a source comment is dropped or duplicated.
- [x] Formatter tests fail if formatted output parses to a different AST.
- [x] Formatter tests fail if formatting is not idempotent.
- [x] Comment fixtures cover top-level, expression, declaration, list, handler, and trailing-comment positions.
