## Description
Move the linked-program model and `link_*.mbt` implementation out of `lanec/module/compile` so executable elaboration can consume linking without depending on LoisVM orchestration.

## Design
Create `lanec/module/link` as a coherent target-independent package. Preserve public behavior and remove the `module/compile` reverse dependency on `lanec/loisvm_lowering`; orchestration may depend on both packages.

## Acceptance Criteria
- [ ] `lanec/module/link` owns Linked Program construction, validation, remapping, identity, and diagnostics.
- [ ] No package cycle or execution-target dependency exists in the link package.
- [ ] Existing compile, link, artifact, and command tests pass unchanged.
- [ ] Generated interfaces reflect only intentional ownership moves.
