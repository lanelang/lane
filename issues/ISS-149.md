# ISS-149: Separate interface artifacts from object artifacts

## Metadata
- Type: task
- Status: closed

- Created: 2026-07-08

## Description
`ModuleObjectArtifact` currently embeds a full `ModuleInterface`. That makes
`.lmo` behave like `object + interface`, even though Lane already has `.lmi`
for full interface metadata. The linker also reconstructs `CompiledModule`
from object artifacts, which keeps checked/source-facing concepts alive in the
link path.

This should be split so `.lmi` remains the full interface artifact and `.lmo`
becomes a link object with only fingerprints, Buslane object payload, exports,
imports, entries, and explicit link metadata.

## Design
Hard switch the artifact schema rather than keeping compatibility layers:

- keep `ModuleInterfaceArtifact` as the full `.lmi` format;
- remove full `ModuleInterface` from `ModuleObjectArtifact`;
- add explicit link-only metadata to object artifacts for data the linker still
  needs, especially runtime effect operation signatures;
- make linking from `.lmo` build link modules directly instead of recreating
  `CompiledModule` with fake checked source/interface data;
- keep fingerprint validation on `.lmo`, but do not require complete interface
  contents to validate an object artifact;
- remove or narrow combined `CompiledModuleArtifact` APIs if they imply that an
  object artifact should carry an interface artifact.

## Acceptance Criteria
- [x] `ModuleObjectArtifact` no longer contains a full `ModuleInterface`.
- [x] `.lmo` binary encode/decode no longer writes or reads `ModuleInterface`.
- [x] Link from object artifacts does not construct fake `CompiledModule` or fake checked source/interface data.
- [x] Runtime effect operation metadata required by `lane run`/`exec` survives through `.lmo` and linked artifacts.
- [x] Tests cover `.lmi` roundtrip, `.lmo` roundtrip without interface payload, link validation, and damaged object metadata.
- [x] Validation passes with `moon check --warn-list +73`, `moon test --warn-list +73`, and `tools/check-lane-run-examples.sh`.
