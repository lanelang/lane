## Description
`modules/lanec/typecheck/probe.mbt` is a transitional abstraction left from the expression type/effect synthesis refactor. The old type-only effect synthesis has been removed, but `probe` still hides two different responsibilities behind one vague name: generic argument inference needs silent type synthesis, while handler checking needs a preliminary arm-effect analysis before final resume types are known.

## Design
Remove `probe.mbt` as a named layer. Generic argument inference should call an explicitly named silent checked-synthesis helper that does not commit diagnostics or checker environment mutations. Handler checking should own its two-phase effect algorithm with a local helper, rather than routing through a generic probe API. Keep behavior unchanged and avoid adding compatibility wrappers.

## Acceptance Criteria
