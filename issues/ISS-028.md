## Description
`Module Input Set` handling supports multiple caller needs: strict compilation of a root plus supplied libraries, root plus import-reachable workspace sources for LSP, and conservative fallback when unrelated workspace files are invalid. These modes are currently expressed by composing lower-level helpers and caller-side fallback behavior, so missing imports, root parse errors, library parse errors, and unrelated workspace parse errors are not represented as an explicit interface.

## Design
Deepen the module input selection and import graph check interface so callers choose an explicit mode such as strict whole-set analysis or import-reachable analysis. Keep Lane module identity source-based, preserve the acyclic import graph rule, and avoid moving filesystem discovery into `lanec`.

## Acceptance Criteria
- [x] The intended error behavior for root source, reachable imports, missing imports, and unrelated workspace files is documented in the issue or implementation notes before code changes.
- [x] CLI compile/check behavior remains strict for user-supplied module inputs.
- [x] LSP workspace analysis can ignore unrelated invalid workspace files without silently losing diagnostics for reachable imports.
- [x] Validation includes module input tests and `moon check --warn-list +73`.

## Notes
- 2026-07-01: Created from architecture review. The motivating friction is that caller-side fallback hides meaningful distinctions between module input error modes.
- 2026-07-01: Intended error policy before code changes: CLI/check/compile uses strict whole-set analysis, so root parse errors, any user-supplied library parse errors, duplicate module inputs, missing imports, and import cycles all produce diagnostics. LSP workspace input selection uses import-reachable analysis, so root parse errors still fail selection, missing imports and cycles reachable from the root still produce diagnostics, reachable library parse errors are returned, and parse errors in unrelated workspace files are ignored instead of causing a caller-side fallback that hides reachable diagnostics.
