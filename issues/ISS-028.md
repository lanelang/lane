## Description
`Module Input Set` handling supports multiple caller needs: strict compilation of a root plus supplied libraries, root plus import-reachable workspace sources for LSP, and conservative fallback when unrelated workspace files are invalid. These modes are currently expressed by composing lower-level helpers and caller-side fallback behavior, so missing imports, root parse errors, library parse errors, and unrelated workspace parse errors are not represented as an explicit interface.

## Design
Deepen the module input selection and import graph check interface so callers choose an explicit mode such as strict whole-set analysis or import-reachable analysis. Keep Lane module identity source-based, preserve the acyclic import graph rule, and avoid moving filesystem discovery into `lanec`.

## Acceptance Criteria
- [ ] The intended error behavior for root source, reachable imports, missing imports, and unrelated workspace files is documented in the issue or implementation notes before code changes.
- [ ] CLI compile/check behavior remains strict for user-supplied module inputs.
- [ ] LSP workspace analysis can ignore unrelated invalid workspace files without silently losing diagnostics for reachable imports.
- [ ] Validation includes module input tests and `moon check --warn-list +73`.
