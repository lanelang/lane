## Description
Implement the schema-v2 image model, codec, formatting, and structural checks for Instance Globals and the optional Instance Initializer.

## Design
Keep schema v1 unchanged. Add dense zero-based GlobalId, global metadata, initializer FunctionId, `init_global`, and `borrow_global` with atomic erased-companion handling as specified by ADR-0114.

## Close Notes
Implemented dense zero-based `GlobalId`, global metadata with immediate erased companions, optional initializer FunctionId, and `InitGlobal`/`BorrowGlobal` tags 0x44/0x45. The encoder preserves byte-for-byte schema-v1 output for images without v2 features and selects schema v2 only for initializer/global content. The BytesView bitstring decoder accepts both schemas, enforces initializer ABI and global-table shape, keeps u32 implementation-limit failures distinct from malformed encodings, and rejects v2 instruction tags in schema v1. Disassembly and README describe both schemas. Interpreter and Wasm paths reject v2 execution explicitly until ISS-192 and ISS-193 implement lifecycle semantics. Verification: stable v1 encoding regression; bytecode codec tests 13/13 on wasm, wasm-gc, js, native; interpreter tests 11/11 on every target; Wasm tests 33/33 native; LoisVM lowering tests 27/27 on every target; artifact-binary tests 4/4 on every target; cross-target `moon check --warn-list +73`; `moon info`; `moon fmt`; `git diff --check`.
