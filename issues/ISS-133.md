## Description
The typechecker currently keeps a type-only expression checker (`check_expr` / `synthesize_expr`) beside the checked-expression builder (`check_checked_expr` / `synthesize_checked_expr`). This duplicates core typing rules for calls, function literals, matches, variants, operations, blocks, and fields. New type system features have to update both tracks, which makes divergence likely and keeps the typecheck code spaghetti-shaped.

## Design
Hard switch expression typing to the checked-expression path. Type-only callers should get facts from checked synthesis/checking results instead of maintaining a parallel implementation. Do not leave compatibility wrappers that preserve a second rule engine.

## Acceptance Criteria
- [x] Type-only expression typing rules are removed or reduced to direct checked-expression adapters with no independent rule logic.
- [x] Existing call, function literal, match, variant, operation, block, and field typing behavior remains covered by tests.
