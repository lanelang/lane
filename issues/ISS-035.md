## Description
The Buslane verifier currently synthesizes only value types. Effect-core
verification needs expression synthesis to also track effects, including calls,
`perform`, handlers, and resume usage.

## Design
Change expression synthesis to produce both a value type and an effect. Verify
calls by combining callee latent effects with argument effects. Verify
`perform` against operation signatures. Verify handlers by checking return
clauses, operation alternatives, resume types, and removal or unification of
handled singleton effects with optional residual effect rows.

## Acceptance Criteria
- [x] Expression synthesis returns value type plus effect.
- [x] Calls combine argument effects and callee latent effects.
- [x] `perform` checks payload types and produces the operation result type
  plus the owning singleton effect.
- [x] Handler verification checks return clauses, operation alternatives,
  resume usage, and handled-effect removal or unification.
- [x] Invalid operation ownership, payload arity, resume arity, and resume type
  cases produce Buslane diagnostics.
- [x] Verifier tests cover closed effect sets, open effect rows, missing
  handlers, wrong operation payloads, and invalid resume types.
- [x] `moon test modules/buslane` passes.
- [x] `moon info && moon fmt` has been run after the change.
