## Description
Lane now has a source-aware diagnostic renderer, but its reports are still much thinner than Rust compiler diagnostics. The main gap is not just formatting: compiler stages often emit shallow string messages instead of structured causes, secondary source locations, concrete facts, and actionable fixes.

## Design
Treat Rust-grade diagnostics as a product of two layers: compiler stages must produce rich diagnostic objects, and presentation targets must render those objects well for CLI, LSP, JSON, and tests. Do not make the output superficially Rust-like without preserving structured information.

## Acceptance Criteria
- [x] Child issues cover diagnostic data modeling, compiler-stage production, renderer behavior, terminal capability policy, and regression tests.
- [x] The resulting CLI diagnostics explain why an error occurred, where the relevant causes are, and what concrete action the user can take.
- [x] LSP diagnostics keep equivalent structured information without coupling to terminal formatting.
