# ISS-117: Warn on unused type and effect parameters

## Metadata
- Type: feature
- Status: closed
- Priority: 2
- Labels: area/typecheck, area/diagnostics, area/lint, agent
- Assignee: codex
- Created: 2026-07-05
- Updated: 2026-07-06
- External ref: none

## Description
Lane should emit a warning when a type parameter or effect parameter is declared but unused in the declaration body or signature where it is bound.

## Design
Analyze resolved type/effect parameter symbols rather than matching names in source text. This should cover generic functions, types, effects, operation-level binders, aliases, and any other generic binder form that introduces type or effect parameters. The warning must account for shadowing and should remain consistent with the kind system.

## Acceptance Criteria
- [x] Unused type parameters produce warning diagnostics at their binder locations.
- [x] Unused effect parameters produce warning diagnostics at their binder locations.
- [x] Used parameters in types, effect sets, operation signatures, and generic applications do not warn.
- [x] Shadowing is handled by symbol identity, not by name matching.
- [x] The behavior is locked by focused tests and diagnostic fixtures.
