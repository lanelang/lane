# ISS-032: Attach latent effects to Buslane function types

## Description
Buslane function types currently only carry parameter and result types. The
effect roadmap requires every function type to carry a latent effect, with pure
functions represented by the empty effect.

## Design
Update the function type shape to include a latent effect. Migrate all pure
function construction sites to use the empty effect, then update equality,
substitution, metadata helpers, pretty output, and tests. Keep the latent
effect as an effect set with optional residual row information, not a flattened
operation set.

## Acceptance Criteria
- [ ] Function types store parameter types, result type, and latent effect.
- [ ] Existing pure Buslane programs still verify by using the empty effect.
- [ ] Type equality and substitution account for latent effects.
- [ ] Pretty output includes latent effects where required without noisy output
  for pure functions unless the chosen text format requires it.
- [ ] `moon test modules/buslane` passes.
- [ ] `moon info && moon fmt` has been run after the change.
