## Description
`modules/lane/main.mbt` currently contains the command workflows for check, compile, fmt, link, runlinked, run, and lsp startup. Several workflows directly combine file IO, library discovery, compiler calls, artifact IO, runtime evaluation, entry lookup, and diagnostic printing. This weakens locality when the separate compilation model, linked program behavior, or command reporting changes.

## Design
Concentrate related Lane command workflows behind deeper modules while preserving the public command behavior. Likely candidates are direct file run, separate compile/link, linked program run, and formatting/checking. Avoid speculative framework work; move only behavior that already has repeated command-level coupling.

## Acceptance Criteria
- [x] Command workflows can be understood and tested in smaller modules than the full `main.mbt` command dispatcher.
- [x] Existing `lane check`, `lane run`, `lane compile`, `lane link`, `lane runlinked`, `lane fmt`, and `lane lsp` behavior remains stable.
- [x] `tools/check-lane-run-examples.sh` passes after any implementation.
