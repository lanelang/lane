## Description
Convert Lane list-like syntax nodes to use the shared width-sensitive formatter primitives.

## Design
Cover function calls, effect operation calls, enum constructor payloads, pattern payloads, function parameters, function type parameters, operation payloads, type parameters, type arguments, operation type witnesses, handler operation type binders, effect sets, struct literal fields, struct pattern fields, and selective import items. Preserve the no-trailing-comma style and do not split source atoms.

## Acceptance Criteria
- [ ] Parenthesized call, operation, constructor, pattern, parameter, and payload lists have inline and broken test coverage.
- [ ] Bracketed type parameter, type argument, type witness, and handler binder lists have inline and broken test coverage.
- [ ] Braced effect set, struct literal, struct pattern, and selective import lists have inline and broken test coverage.
- [ ] Formatter output remains parseable and idempotent for each covered shape.
- [ ] `moon test modules/lanec/syntax --warn-list +73` and `moon test modules/lanec/format --warn-list +73` pass.
