## Description
Checked Lane operation-call expressions must lower to Buslane `perform`
expressions, not ordinary value calls or source-shaped placeholders.

## Design
Use the operation metadata mapping from ISS-048 and the effect/type lowering
from ISS-049 to lower checked operation calls into Buslane `Perform` nodes.
Reuse the existing expression lowering order for payload expressions so
evaluation order and expression effects remain explicit in Buslane.

## Acceptance Criteria
- [ ] Checked Lane operation-call expressions lower to Buslane `perform`.
- [ ] Payload expressions lower in the same left-to-right order as ordinary
  call arguments.
- [ ] Operation identity is resolved through the lowering metadata mapping.
- [ ] Operation calls are not lowered as ordinary function calls.
- [ ] Focused integration snapshots cover Lane operation calls through Buslane
  `perform`.
- [ ] `moon test modules/lanec/elaborate --warn-list +73` passes.
- [ ] `moon check modules/lanec --warn-list +73` passes.
- [ ] `moon info && moon fmt` has been run after the change.
