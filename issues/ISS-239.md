## Description
Buslane owns `TypeSubstitution`, `Type::substitute`, and `Effect::substitute`, but LoisVM lowering maintains a second `TypeArgumentBinding` representation and private recursive type/effect substitution. The implementations differ in lookup order, chained replacement behavior, kind-mismatch handling, and function-effect traversal. Lowering correctness therefore depends on undocumented stage invariants instead of one substitution contract.

## Design
Deepen the Buslane substitution module so it explicitly supports the composition, shadowing, and validation semantics needed by LoisVM lowering. Convert lowering bindings to that representation once, then delete the duplicate recursive substitution implementation. Preserve explicit lowering errors for type/effect kind mismatches rather than silently ignoring invalid replacements. Document whether latent effects are expected to remain at this stage and enforce that invariant.

## Acceptance Criteria
- [ ] LoisVM lowering uses the Buslane-owned substitution implementation for types and effects.
- [ ] Chained substitutions, binder shadowing, and latest-binding behavior have explicit tests.
- [ ] Type-for-effect and effect-for-type mismatches remain structured lowering errors.
- [ ] Function parameter, result, and latent-effect traversal has one documented behavior.
- [ ] The private `substitute_type` and `substitute_effect` implementations in LoisVM lowering are removed.
- [ ] `moon info`, `moon fmt`, focused tests, full tests, examples, and `git diff --check` pass.
