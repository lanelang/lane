## Description
Rust diagnostics are useful because they connect the error site to definitions, conflicting declarations, expected-type origins, actual-type origins, and other related locations. Lane diagnostics often have only one primary span and encode the rest as plain text.

## Design
Audit common resolver, typechecker, pattern, effect, and handler diagnostics. For each high-value class, attach secondary labels or related diagnostics for definitions, conflicts, expectation origins, and inferred values. Keep related information structured so LSP can show it without parsing terminal output.

## Acceptance Criteria
- [x] Unbound name, duplicate definition, type mismatch, handler mismatch, and missing field diagnostics include useful secondary spans where available.
- [x] LSP diagnostics expose related information for the same cases.
- [x] Golden tests assert the rendered related locations for representative examples.
