## Description
Remove top-level and local aliases, trivial beta redexes, dead pure bindings, and constructor-match redexes before initializer materialization and ANF lowering.

## Design
Implement a terminating simplification fixpoint inside the effect-aware core optimizer. Preserve evaluation order and never remove or duplicate effectful expressions. Rewrite direct aliases such as `let value#33 = value#2` at their uses so compiler intrinsics and runtime imports do not acquire synthetic instance globals.

Reuse existing single-constructor match semantics rather than adding a second incompatible product-elimination rule.

## Acceptance Criteria
- [x] Reachable direct aliases are substituted without creating instance globals.
- [x] Trivial local lets and beta redexes are removed without reordering effects.
- [x] Known-constructor matches select the matching arm when binders can be substituted safely.
- [x] Dead pure definitions disappear while observable expressions remain.
- [x] The Coroutine Scheduler core no longer contains the Basic external alias chain.
