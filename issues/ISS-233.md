# ISS-233: Preserve comments in lists and delimiters

## Description
Comments around selective import items, parameters, call arguments, type arguments, commas, opening delimiters, and closing delimiters are not represented by the current pretty-printer hooks and are silently dropped.

## Design
Make list and delimiter combinators trivia-aware. Preserve comments before items, after separators, immediately after opening delimiters, and immediately before closing delimiters in both inline and broken layouts. Comments force a broken layout when inline rendering cannot preserve their line attachment.

## Acceptance Criteria
- [x] Selective imports preserve item and delimiter comments.
- [x] Function parameters and function-literal parameters preserve comments.
- [x] Call arguments, type arguments, payloads, patterns, and struct fields preserve comments.
- [x] Separator and closing-delimiter comments survive and formatting is idempotent.
- [x] Width-sensitive tests cover both inline and forced-broken layouts.

## Close Notes
- 2026-07-18: Added token-matched delimiter trivia queries and made import, declaration, type, call, literal, payload, pattern, handler, and block list printers consume comments within their exact paired delimiters. Pre-comma line comments now move after the comma so the separator remains parseable. Formatter regressions cover comments before items, after separators, before closes, generic witnesses, function types, and narrow layouts; `moon test modules/lanec/format` passes 40/40.
