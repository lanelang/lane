# ISS-224: Devirtualize calls and eliminate nonescaping closures

## Metadata

## Description
Convert statically known callable invocations to direct calls and eliminate closure environments that do not escape their immediate call region.

## Design
Use core occurrence facts where available and VM CFG def-use and escape analysis after closure lowering. Replace known `call_value` and tail-call-value sites with direct forms. Scalar-replace nonescaping environments by threading captures directly or through an explicit direct-call environment only when allocation can be proven unnecessary.

Do not change first-class callable representation or assume that reusable continuations are single-shot.

## Acceptance Criteria
- [x] Known callable targets use direct call or direct tail-call instructions.
- [x] Immediately invoked nonescaping closures avoid callable and environment allocation.
- [x] Escaping closures and reusable continuations retain ordinary closure semantics.
- [x] ARC ownership transfers remain valid after devirtualization.
- [x] The Coroutine Scheduler emits fewer `make_env`, `make_closure`, and indirect-call instructions.

## Notes
- 2026-07-17: Started after ISS-223 established the pre-ARC CFG simplification boundary and conservative trivial-slot allocator.
- 2026-07-17: The baseline contains 136 environment allocations, 136 closure constructions, and 199 indirect calls or tail calls.
- 2026-07-17: An initially broader unique-reference environment ABI rewrite caused parser-combinator nontermination and an Optics Wasm indirect-call signature mismatch. The corrected proof requires a unique target reference and a leaf target body; higher-order, recursive, and shared targets keep their environment ABI.

## Close Notes
Added pre-ARC known-callable devirtualization for adjacent `const_function`/`make_closure` plus call or tail-call sequences, guarded by whole-body value-use counts so escaping and reused callables remain first class. Added a stricter allocation-elimination path for immediately invoked environments: only a uniquely referenced leaf lifted function is rewritten in place to receive captures as hidden owned arguments and witnesses as hidden witness inputs, avoiding function cloning and code-size regressions. The Coroutine Scheduler drops from the recorded 136 environments, 136 closures, and 199 indirect calls/tail calls to 108 environments, 105 closures, and 103 indirect calls/tail calls. Verification: VM CFG 17/17, LoisVM lowering 35/35, interpreter 21/21, Wasm 45/45, Lane 56/56, all runnable examples including parser combinators and Optics on JIT and interpreter paths, Scheduler output `1112211322`, warning-enabled check, and `git diff --check`.
