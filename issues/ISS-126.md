## Description
Parser wording is already partly owner-local through `ParseErrorKind::diagnostic_*`, but compile and format still build parser diagnostics separately. Consolidate parser conversion so parser-owned text feeds the shared typed diagnostic API consistently.

## Design
Keep parser text rules near `ParseErrorKind`. Add or adapt a parser diagnostic spec/conversion that both compile and format can consume. Avoid making parser depend on source-language compile orchestration.

## Acceptance Criteria
- [x] Compile and format parser diagnostics use one parser-owned conversion path.

- [x] Parser recovery diagnostics in invalid examples keep the improved semantic text.
- [x] Formatter parser diagnostic tests pass.
- [x] `tools/check-lane-run-examples.sh` passes.
