## Description
Create the shared formatting primitives and test helper needed before converting individual Lane syntax nodes to width-sensitive output.

## Design
Refactor `modules/lanec/syntax/pretty.mbt` so list delimiters, separated lists, assignments, signatures, and chains can share a small set of internal helper functions. Add a formatter test helper that checks expected output, parse-back, and idempotence. The helper should allow a small internal width such as 40 or 60 to force broken layouts in focused tests.

## Acceptance Criteria
- [ ] Shared primitives cover parenthesized, bracketed, and braced comma lists with inline and broken modes.
- [ ] Broken lists use one item per line, two-space indentation, independent closing delimiter lines, and no trailing comma.
- [ ] Test helper asserts formatted output, parse-back, and idempotence.
- [ ] Existing formatter tests keep passing or are updated only for intentional formatting changes.
- [ ] `moon test modules/lanec/syntax --warn-list +73` and `moon test modules/lanec/format --warn-list +73` pass.
