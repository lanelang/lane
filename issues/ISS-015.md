## Description
Struct literal/pattern and public interface diagnostics should guide the user toward a concrete source edit. Current messages such as `unknown_field x`, `missing_field y`, `duplicate_type_member T`, `hidden_type_escape ...`, and `private type `Hidden` in public interface` do not explain the owner type, declaration context, or repair choices.

Current message inventory:

- `unknown_field ...`
- `missing_field ...`
- `duplicate_field ...`
- `unknown_type_member ...`
- `missing_type_member ...`
- `duplicate_type_member ...`
- `duplicate_type_parameter ...`
- `hidden_type_escape ...`
- `private type ... in public interface`

## Design
Report the affected owner type or public item whenever available. For private type escapes, use Rust-like wording: the public item exposes a private type; label the public item and the private type reference; help should suggest making the type public or changing the public signature to avoid exposing it.

## Acceptance Criteria
- [x] Field diagnostics name the struct type and label the field occurrence.
- [x] Missing field/type-member diagnostics list the required member and the construction pattern/literal being checked.
- [x] Duplicate field/type-member/type-parameter diagnostics label both duplicate occurrences when spans are available.
- [x] Private type escape diagnostics include primary label, explanation, and repair options.
- [x] Golden tests cover public value, function, struct field, enum payload, and alias private type escapes.
