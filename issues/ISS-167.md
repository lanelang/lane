## Description
Add an end-to-end conformance suite proving that effectful Lane programs compile into effect-erased bytecode and execute equivalently in the LoisVM interpreter and Wasmoon-backed Wasm tier.

## Design
Use the Buslane interpreter as the source-semantics oracle where practical, then execute the same lowered image through both LoisVM backends. Inspect complete observable results or output traces rather than asserting rendered substrings. Include ownership-sensitive cases so continuation closure capture and repeated resume exercise ARC behavior.

## Acceptance Criteria
- [x] Tests cover direct and nested deep handlers.
- [x] Tests cover final clauses, handler-arm effects, and nearest-handler precedence.
- [x] Tests cover single and multi-shot resume with captured reference-counted values.
- [x] Tests cover generic handlers or continuations carrying layout witnesses.
- [x] Tests cover operations that escape to the primitive runtime host boundary.
- [x] Interpreter and Wasm outputs agree, and emitted bytecode contains no effect-specific representation.
- [x] Full `moon test --target native` and `moon check --target all --warn-list +73` pass.

## Close Notes
Added a native end-to-end source corpus in `modules/lane/loisvm_effect_conformance_wbtest.mbt`. Every case compiles through the production Lane-to-LoisVM pipeline, asserts that the only residual effect boundary is the ordinary primitive `lane.effect.v1/Basic.Io/Write/println` runtime import, and compares complete output traces from the LoisVM interpreter and Wasmoon-backed Wasm tier. The corpus covers direct and nested deep handlers, inner-arm dispatch to the nearest outer handler, final clauses, single- and multi-shot resume over captured `String` values, and generic operation resumptions carrying erased layout witnesses. `Instruction` and `Terminator` remain effect-free by construction in `modules/loisvm/bytecode/model.mbt`.
