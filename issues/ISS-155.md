## Description
Lane formatting must become a complete trivia-preserving architecture. Formatting should preserve `//` comments and meaningful blank lines while continuing to canonicalize ordinary whitespace through width-sensitive pretty printing.

## Design
Follow ADR 0065. Introduce concrete syntax sidecars from the lexer, return parsed source through a parser domain enum, derive formatter-local trivia views from gap-indexed trivia, and migrate the formatter to use trivia-aware pretty printing. Do not add comment fields to semantic AST nodes, do not implement post-render comment reinsertion, and do not add range or error-tolerant formatting in v1.

## Acceptance Criteria
- [x] Concrete syntax sidecars carry source text, tokens including EOF, and gap-indexed trivia.
- [x] Parser APIs expose parsed and failed states without optional payload invariants.
- [x] Formatter preserves comments and meaningful blank lines without losing width-sensitive formatting.
- [x] Formatter verification covers comment no-loss, AST roundtrip equivalence, and idempotence.
