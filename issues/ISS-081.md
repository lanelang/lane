## Description
Implement cursor-context classification for completion so candidate collection can distinguish ordinary identifier, module-qualified, nominal/effect owner, receiver selector, and import declaration contexts.

## Design
Build a small completion-context classifier that determines prefix text, replacement range, expected namespace, and qualified owner shape. Handle manual completion and LSP trigger characters `.` and `:`. Treat a visible module binding on the left of `.` as module-qualified completion before considering receiver selector completion. Keep parser recovery narrow in v1: identifier prefixes, qualified access prefixes, and import declaration module paths.

## Acceptance Criteria
- [x] The classifier returns replacement ranges that cover the current identifier prefix.
- [x] `.` contexts distinguish module binding access from receiver selector access.
- [x] `::` contexts recognize nominal owners and effect owners, including module-qualified owners.
- [x] Import declaration contexts recognize module path prefixes.
- [x] Tests cover supported contexts and unsupported cursor positions.
