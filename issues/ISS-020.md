## Description
Diagnostic rewrites are easy to regress because tests often assert only the internal diagnostic kind or the first message string. Add golden tests that lock down user-facing Rust-style output across CLI and LSP entry points.

## Design
Create focused fixtures rather than relying on large example output churn. Cover both CLI pretty output and LSP JSON diagnostics. Snapshot tests are acceptable for multi-line reports; stable assertion tests are better for diagnostic codes and structural fields.

## Acceptance Criteria
- [x] Golden tests cover at least one diagnostic from each child issue family.
- [x] Tests assert diagnostic code and headline separately from full pretty text where possible.
- [x] CLI snapshots include labels, notes, and help text.
- [x] LSP tests verify code, range, severity, message, and related information where used.
- [x] `moon test --target native --warn-list +73` passes after snapshots are updated.
