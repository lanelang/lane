## Description
Lane should support `Double` as a primitive IEEE 754 binary64 type. Source literals containing a decimal fraction or exponent should type as `Double`, remain distinct from `Int` literals, and lower through checked IR, Buslane, artifacts, and interpreter values without relying on Basic library definitions.

## Design
Add `Double` to the source and Buslane primitive type model. Extend lexer/parser/source AST with `Double` literals using the agreed syntax: allow forms such as `1.0`, `0.5`, `1e3`, `1e-3`, and `1.0e+3`; reject `.5`, `1.`, `inf`, and `nan`. Parse literals to finite binary64 values while retaining original source spelling for diagnostics; reject overflow such as `1e309` and nonzero underflow to zero such as `1e-4000`, while allowing representable subnormals.

Support `Double` literal patterns as refutable patterns using floating equality, without adding an exhaustive primitive pattern set. Lower checked `Double` literals to Buslane literals. Buslane text, pretty, artifact inspect, parser roundtrip, verifier, and interpreter values should use stable canonical roundtrip decimal text for semantic values.

Non-goals: Basic `%f64_*` operations, Basic special constants, printing/conversion helpers, and implicit or explicit `Int`/`Double` conversions.

## Acceptance Criteria
- [x] Valid examples cover `Double` annotations, decimal and exponent literals, representable subnormal literals, and `Double` literal patterns.
- [x] Invalid examples reject `.5`, `1.`, `1e309`, `1e-4000`, `let x : Double = 1`, and `let x : Int = 1.0`.
- [x] Buslane text/artifact roundtrip preserves `Double` literals with canonical text.
- [x] `../spec` documents the `Double` primitive, literal syntax, finite literal rule, and pattern semantics.
- [x] `moon check --warn-list +73`, `moon test --warn-list +73`, `tools/check-lane-run-examples.sh`, and `moon info && moon fmt` pass.
