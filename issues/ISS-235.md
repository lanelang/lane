## Description
The formatter test helper checks expected text, comment text order, reparsing, and idempotence, but does not compare the original and formatted syntax or verify comment ownership. AST-only public formatter functions also cannot preserve concrete comments, while their names do not make that limitation explicit. Pure blank-line trivia is recorded by the lexer but discarded by the formatter view.

## Design
Add structured syntax-equivalence and trivia-consumption oracles, cover pure blank-line policy explicitly, and remove or narrow misleading AST-only public formatting APIs. Update ADR and issue close notes to describe the implemented contract rather than claiming unsupported fidelity.

## Acceptance Criteria
- [x] Tests compare normalized parsed syntax before and after formatting.
- [x] Tests inspect complete trivia attachment/consumption results.
- [x] Meaningful blank-line behavior is specified and tested independently of comments.
- [x] AST-only formatting APIs are private, removed, or explicitly named as canonical AST rendering.
- [x] ADR-0065 matches the final implementation.
