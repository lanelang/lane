## Description
The current Buslane interpreter is a direct evaluator for the pure core. Deep
handlers require explicit runtime resume values and captured continuations so
`perform` can dispatch to the nearest handler and resume under the same
handler.

## Design
Move interpreter internals to an explicit evaluation-continuation model while
preserving the public API where possible. Represent resume values as captured
continuations under a deep handler. Support nearest-handler dispatch,
deep-handler reinstallation on resume, multi-shot resume, strict left-to-right
evaluation, and runtime errors for unhandled `perform`.

## Acceptance Criteria
- [x] Interpreter internals can capture and invoke continuations for resume
  values.
- [x] `perform` dispatches to the nearest matching handler table entry.
- [x] Invoking a resume value reinstalls the same deep handler.
- [x] Multi-shot resume behavior is covered by tests.
- [x] Unhandled `perform` produces a Buslane runtime error.
- [x] Interpreter tests cover direct handling, nested handlers, resumed
  continuations, multi-shot resume, and unhandled operations.
- [x] `moon test modules/buslane/interpreter` passes.
- [x] `moon info && moon fmt` has been run after the change.
