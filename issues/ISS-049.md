## Description
Lane checked types carry effect terms and function latent effects. Buslane
lowering must translate those effect terms into the Buslane `Effect` model so
function types and future `perform`/handler expressions verify against the same
effect identities.

## Design
Add a lowering helper for checked Lane effect terms that covers empty effects,
singleton effects, unions, and effect row variables. Use it when lowering
function types and named function signatures. Preserve pure functions as the
Buslane empty effect and reuse the metadata mapping introduced by ISS-048.

## Acceptance Criteria
- [x] Lane empty effects lower to Buslane `Effect::Empty`.
- [x] Lane singleton effects lower to Buslane effect singletons via the
  metadata mapping.
- [x] Lane unions and effect row variables lower to Buslane effect unions and
  kind-`Effect` type parameters.
- [x] Function types and named function declarations lower their latent
  effects.
- [x] Existing pure programs still lower and verify as pure.
- [x] Focused lowering snapshots or tests cover pure and non-empty latent
  effects.
- [x] `moon test modules/lanec/elaborate --warn-list +73` passes.
- [x] `moon check modules/lanec --warn-list +73` passes.
- [x] `moon info && moon fmt` has been run after the change.
