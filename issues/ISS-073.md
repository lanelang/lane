## Description
Extend top-level type alias syntax so the alias right-hand side can represent an effect-level expression, including effect set literals like `{ Console, Log }`, without treating value blocks as type-level forms.

## Design
Introduce a kinded alias body shape for alias RHS parsing and pretty-printing. Ordinary type positions remain ordinary type syntax, and effect annotations remain effect-term syntax. The alias RHS is the only new top-level position that needs to accept either ordinary type-level expressions or effect-level expressions.

## Acceptance Criteria
- [ ] Parser accepts `type UI = { Console, Log }`.
- [ ] Parser accepts generic effect alias bodies such as `type Task[E : Effect] = { Coroutine, E }`.
- [ ] Formatter prints short effect alias bodies inline and breaks long bodies predictably.
- [ ] Parser rejects value-expression blocks as alias bodies.
