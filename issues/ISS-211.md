## Description
Wire the LoisVM interpreter to the Host Object Table and direct-value runtime-import ABI, including ARC wrapper allocation, borrowed argument resolution, owned results, and execution shutdown.

## Design
Use the shared runtime model rather than embedding host-language objects in `VMValue`. Preserve callee-owned argument behavior, String borrow/copy rules, fatal host failure, and normal execution-instance cleanup.

## Acceptance Criteria
- [x] Interpreter runtime imports accept and return `Opaque`.
- [x] Copies and aggregate/closure storage retain the ordinary Lane wrapper.
- [x] Normal release and instance shutdown finalize objects with no double finalization.
- [x] Host panic, invalid handle, exhausted object limit, and use-after-close behavior are covered.
- [x] Existing primitive runtime-import behavior remains compatible.
