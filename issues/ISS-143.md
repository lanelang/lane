## Description
Buslane should own binary serialization for Buslane core structures used inside Lane artifacts. The compiler artifact codec should delegate Buslane programs and related core data to a Buslane codec package instead of embedding Buslane text or duplicating Buslane AST encoding in `lanec`.

## Design
Create `modules/buslane/codec` depending on `bytecodec`. Encode and decode Buslane `Program`, `TopTerm`, `MetadataRegistry`, core identities, types, effects, expressions, handler structures, and other data needed by module object and linked program artifacts.

Use a Buslane codec version distinct from Lane artifact container and artifact schema versions. Decoding should use typed `suberror`s and reject malformed tags, invalid values, and trailing bytes. Buslane text and pretty printing remain for readable core syntax, tests, and inspect output; they do not participate in artifact loading.

Non-goals: CLI diagnostics, Lane artifact container parsing, artifact schema fields, or replacing Buslane text/pretty.

## Acceptance Criteria
- [x] `modules/buslane/codec` exists and depends on `modules/bytecodec`.
- [x] Buslane core structures needed by `.lmo` and `.lbp` can roundtrip through binary codec.
- [x] Malformed binary tests cover invalid tags and trailing bytes.
- [x] Buslane text/pretty tests still pass and remain independent from binary codec.
- [x] `moon info && moon fmt` and relevant Buslane tests pass.
