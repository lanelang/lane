## Description
Add the initial conventional `Stdlib` module implementation in the sibling `../stdlib` project. The module defines the effect and alias shapes that `lane run` and `lane runobj` may recognize when users explicitly compile and link the Stdlib artifacts.

## Design
Provide a normal Lane module named `Stdlib`; do not rely on compiler or command-line implicit injection. The initial public surface should include:

```lane
pub effect Write {
  println(String) -> Unit
}

pub type Console = { Write }
pub type Io = { Console }
```

Do not add `Read`, `File`, `Network`, formatting, stderr, flush, or exception modeling in this issue.

## Acceptance Criteria
- [x] `../stdlib` contains a `Stdlib` module with `Write.println(String) -> Unit`, `Console`, and `Io`.
- [x] The Stdlib module can be compiled manually to `.lmi` and `.lmo`.
- [x] No compiler or command tooling implicitly injects the Stdlib module.
- [x] Examples or docs show explicit compile/link input usage.
