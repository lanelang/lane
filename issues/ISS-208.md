## Description
Carry External Effect and External Type flavor through Buslane metadata, module interfaces, object artifacts, linking, executable specialization, inspection, and codecs until LoisVM lowering.

## Design
Keep flavor orthogonal to nominal identity and kind. `Io` resolves to the same External Effect flavor with compiler-provided origin. Preserve source External Type identity in typed IR and artifacts, but do not emit that identity into final LoisVM bytecode.

## Acceptance Criteria
- [ ] Buslane metadata explicitly represents external type/effect flavor.
- [ ] Text and binary codecs roundtrip complete flavored metadata.
- [ ] Module interface, object artifact, linker remapping, and executable passes preserve flavor.
- [ ] External Effects remain residual and are erased only at the existing residual-erasure boundary.
- [ ] Complete-value inspect tests cover flavor and linking behavior.
