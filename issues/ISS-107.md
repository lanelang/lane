# ISS-107: Support explicit generic applications on ordinary expressions

## Description
Lane supports generic declarations and recently gained Type-kind and Effect-kind generic arguments through effect owners and constructors, but ordinary function expressions still cannot be explicitly instantiated. This prevents source forms such as `id[Int](1)`, `id[Int]`, and `yield[Log]()` even though the checked IR, ANF, and Buslane lowering already have type-application nodes.

## Design
Add expression-level generic application syntax `expr[Arg, ...]` that produces an explicit type application before ordinary call checking. Reuse the existing kind-aware generic argument checker so Type and Effect parameters are validated uniformly. Keep operation-call syntax separate: `Effect[T]::op!()` remains the effect operation form, while `f[T](...)` is ordinary function application.

## Acceptance Criteria
- [x] Parser, syntax pretty-printer, resolver, desugarer, and checked pretty-printer preserve ordinary expression type applications.
- [x] Typechecking accepts explicit Type and Effect arguments on generic function values and rejects arity mismatch, kind mismatch, non-generic callees, and empty `[]`.
- [x] Explicitly instantiated function values can be called and can also be assigned as first-class values when their instantiated type matches the expected type.
- [x] Valid examples demonstrate ordinary explicit generic calls and update the coroutine example to use the ideal `yield[Log]()` wrapper.
- [x] Invalid examples cover the rejected forms and fixture expectations are updated.
- [x] `moon check --warn-list +73`, `moon test --warn-list +73`, `tools/check-lane-run-examples.sh`, `moon fmt && moon info`, and `git diff --check` pass.

## Close Notes
Implemented explicit ordinary expression type application syntax and typechecking. `expr[Arg, ...]` now parses as a first-class expression node, resolves Type-kind and Effect-kind generic arguments, desugars through to checked `TypeApplication`, and lowers through the existing checked/ANF/Buslane path. Added a dedicated `TypeArgumentsOnNonGeneric` diagnostic instead of reporting non-generic explicit instantiation as a callability error. Updated the coroutine scheduler example to call `yield[Log]()` and added valid/invalid example coverage plus parser/typecheck regressions.
