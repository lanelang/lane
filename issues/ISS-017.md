## Description
Compile/link/artifact diagnostics currently look like linker assertions rather than user guidance. Examples include `missing import Lib`, `import cycle A`, `module interface artifact fingerprint mismatch Lib`, `unsupported module object artifact schema 2 for Lib`, and `missing exported value Lib::value imported by Main`.

Current message inventory:

- `no root source module`
- `duplicate module input ...`
- `duplicate module interface artifact ...`
- `unsupported module interface artifact schema ...`
- `module interface artifact name/fingerprint/module fingerprint mismatch ...`
- `missing import ...`
- `import cycle ...`
- `imported interface fingerprint mismatch Main -> Lib`
- `duplicate module object artifact ...`
- `unsupported module object artifact schema ...`
- `module object artifact interface name/fingerprint/module fingerprint mismatch ...`
- `missing object ...`, `missing imported object ...`
- `module object mismatch ...`, `module fingerprint mismatch ...`
- `imported reference fingerprint mismatch ...`
- `imported reference mismatch ...`
- `missing exported value/type/constructor/field selector ... imported by ...`
- `unsupported linked program artifact schema`
- `invalid linked Buslane program`
- `duplicate linked module ...`
- `linked entry has unknown value ...`
- `duplicate linked entry ...`

## Design
Treat module/artifact errors as build diagnostics with artifact paths, module names, expected/actual fingerprints or schema versions, and next actions. Help should distinguish stale artifacts from incompatible compiler versions, missing compile inputs from missing imports, and object/interface mismatches from source errors.

## Acceptance Criteria
- [ ] Missing import/object diagnostics state which module requested the dependency and which CLI input is missing.
- [ ] Import cycle diagnostics print a cycle path, not only one module name.
- [ ] Schema diagnostics show expected and actual schema version and suggest rebuilding with the current compiler.
- [ ] Fingerprint mismatch diagnostics identify stale interface/object artifacts and name the importer/importee.
- [ ] Link export diagnostics explain that an object file does not match its interface and suggest recompiling the provider module.
- [ ] Golden tests cover compile interface inputs, stale artifacts, missing objects, cycles, and linked program validation.
