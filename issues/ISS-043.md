## Description
After source syntax exists, handler expressions need resolved operation names
and scoped binders for payload patterns, resume continuations, and final
results. Desugaring must preserve that semantic structure for type checking.

## Design
Thread handler expressions through resolve and desugar. Resolve operation names
through the same effect-operation namespace used by operation calls. Introduce
scopes for operation-arm payload pattern binders, resume binders, and final
branch binders without changing type checking yet.

## Acceptance Criteria
- [ ] Resolved handler data uses resolved effect operation names for every
  operation arm.
- [ ] Payload pattern binders, resume binders, and final branch binders are
  scoped and symbolized consistently with existing pattern and value binding
  rules.
- [ ] Desugared handler data preserves resolved operations, payload patterns,
  binders, branch bodies, and source locations needed by diagnostics.
- [ ] Resolve/desugar pretty output covers handler expressions.
- [ ] Tests or snapshots cover qualified, shorthand, and unqualified operation
  arm names where supported by the source syntax.
- [ ] `moon check modules/lanec --warn-list +73` passes.
- [ ] `moon info && moon fmt` has been run after the change.
