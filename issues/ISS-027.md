## Description
`lane lsp` currently concentrates JSON-RPC handling, document storage, URI/path conversion, workspace file scanning, root/import closure selection, fallback analysis behavior, and diagnostic publication in `modules/lane/lsp_core.mbt`. The resulting module has a broad interface: maintainers must understand transport details and compiler-analysis input rules together before changing workspace diagnostics, hover, definition, or inlay hint behavior.

## Design
Introduce a deeper workspace analysis module for `lane lsp` that owns the document-to-analysis-input workflow. Keep filesystem and URI behavior in the Lane command layer, keep target-independent semantic analysis in `lanec`, and avoid moving JSON-RPC transport concerns into compiler packages.
