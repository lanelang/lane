## Description
`modules/lanec/compile/module_compile_*.mbt` is a cohesive file family that is
already behaving like a hidden package inside `lanec/compile`. This is the same
package-boundary smell as the previous `artifact_binary_*` and
`buslane_lower_*` splits: the shared prefix is not just a naming problem; it
means the implementation has grown a local namespace that should be made
explicit.

This issue is not about claiming the code has incoherent responsibilities.
The primary problem is that the module compilation pipeline, its helper model,
and its internal phases are hidden behind filename prefixes instead of a real
MoonBit package boundary.

## Design
Extract the `module_compile_*` family into an explicit package if the dependency
direction stays clean. The intended shape is:

- keep `Milky2018/lanec/compile` as the user-facing compile/check/link facade;
- move the cohesive module compilation implementation behind a new package
  boundary;
- expose only the small API that the compile facade needs;
- avoid legacy re-export layers and compatibility wrappers;
- if shared model types would create cycles, move them to the right common
  package instead of duplicating them.

The split should preserve behavior and avoid rewriting algorithms merely because
files move.

## Acceptance Criteria
- [x] The `module_compile_*` file family no longer lives as a prefix namespace inside `lanec/compile`, or a concrete blocker is documented.
- [x] `lanec/compile` remains a small facade over compile/check/link entrypoints.
- [x] The new package exposes a narrow interface and introduces no cyclic package dependency.
- [x] Existing module compilation, interface-artifact, link, and runtime-effect validation tests still pass.
- [x] Validation passes with `moon check --warn-list +73`, `moon test --warn-list +73`, and `tools/check-lane-run-examples.sh`.
