## Description
Implement `monadic-lift` after open-context resolution so every local selective-CPS bind continuation becomes an ordinary lifted function and closure accepted by the effect-free Buslane pipeline.

## Design
Compute each bind continuation's free values and ordinary type parameters and lift it into a compiler-generated function with explicit captures. Lane does not generate Koka-style yielding and inline variants because it has no global yielding side channel; pure functions already retain the direct fast path, while monadic functions use explicit answer-type CPS. Expose captures to the existing closure lifting, ownership analysis, and ARC insertion stages.

## Acceptance Criteria
- [ ] Every resumable bind continuation has a lifted compiler-generated function.
- [ ] Pure functions remain direct style and monadic functions use explicit answer-type CPS.
- [ ] Lifted functions receive free values and type parameters explicitly.
- [ ] Multi-shot resume creates safe reusable ownership paths under ARC.
- [ ] Generic continuation captures preserve all required layout witnesses.
- [ ] Tests cover nested binds, captured values, generic captures, repeated resume, and ownership-sensitive continuation lifetimes.
