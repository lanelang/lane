## Description
`modules/lanec/module_compile`, `modules/lanec/module_frontend`, and
`modules/lanec/module_interface` are currently top-level sibling packages. They
belong to the same module subsystem, but the flat layout hides their intended
boundary and makes the `lanec` package root noisier than necessary.

## Design
Move the three packages under a single module subsystem directory:

- `modules/lanec/module/frontend`
- `modules/lanec/module/interface`
- `modules/lanec/module/compile`

Preserve the existing code aliases (`@module_frontend`, `@module_interface`,
`@module_compile`) so this remains a package-layout refactor, not a call-site
rename. Keep dependency direction explicit:

- `module/interface` is a model package and must not depend on `module/frontend`
  or `module/compile`;
- `module/frontend` owns source inputs and module graph construction and must
  not depend on `module/compile`;
- `module/compile` may depend on `module/frontend` and `module/interface`;
- downstream packages should depend on the narrowest package they need.
