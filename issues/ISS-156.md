## Description
The lexer currently exposes parser tokens but does not provide the concrete syntax sidecars needed by a trivia-preserving formatter. Add a concrete lexing result that records source text, tokens including EOF, and gap-indexed trivia.

## Design
Extend lexer-owned data with concrete tokens and classified trivia facts. Trivia should store kind and source span referencing the concrete source text, not copied comment text. The token stream should include EOF as a non-printing anchor for final trivia. Lexical errors should prevent concrete syntax from being treated as reliable.

## Acceptance Criteria
- [x] Concrete lexing output includes source text, tokens including EOF, and gap-indexed trivia.
- [x] `//` comments and meaningful blank lines are classified as trivia.
- [x] Ordinary spaces and line breaks are represented only as trivia facts needed for attachment/canonicalization.
- [x] Existing parser token consumers continue to receive a trivia-free token view.
