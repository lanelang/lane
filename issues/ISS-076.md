# ISS-076: Reject duplicate effect and type alias names

## Description
Reject modules that declare an effect and a type or type alias with the same top-level name, avoiding ambiguous effect annotation lookup such as `! IO`.

## Design
Add an explicit cross-namespace duplicate check instead of relying on symbol registry insertion order. The language should not define a priority between effect declarations and effect set aliases. A duplicate top-level effect/type name is an error.

## Acceptance Criteria
- [ ] `effect IO { ... }` and `type IO = ...` in the same module report a duplicate declaration diagnostic.
- [ ] `effect IO { ... }` and `struct IO { ... }` or `enum IO { ... }` are likewise rejected.
- [ ] The diagnostic explains that effect names and type/type alias names cannot overlap.
- [ ] Existing valid effect and type declarations continue to resolve normally.
