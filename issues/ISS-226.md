## Description
Clean up redundant retain, move, and release sequences after VM CFG ownership finalization without changing the canonical LoisVM ownership contract.

## Design
Keep ARC insertion correct and conservative, then run a local ownership-aware cleanup inside the compiler finalization layer. Cancel retain/release pairs only when no intervening escape, alias, call, branch transfer, or destructor-visible action can observe the additional owner. Prefer rewriting pre-finalization ownership flow when a peephole cannot prove safety locally.

## Acceptance Criteria
- [x] Proven redundant retain/release pairs are removed.
- [x] Moves and edge transfers do not create duplicate owners unnecessarily.
- [x] Cleanup works across simplified straight-line blocks without crossing unsafe calls or joins.
- [x] Bytecode ownership tests still reject leaks, double releases, and invalid transfers.
- [x] The Coroutine Scheduler emits fewer explicit ARC operations.

## Notes
- 2026-07-17: Started after ISS-225 converted dead-owner aggregate reads into ownership-transferring projections.
- 2026-07-17: The baseline contains 204 `retain_copy` and 169 `release` instructions.
- 2026-07-17: Inspection of the remaining final bytecode found no safe local retain/release cancellation: the 18 retains represent distinct continuing ownership demands, often one value entering a new environment while another owner continues into a call. The optimization therefore stays in pre-ARC ownership flow instead of adding an unsafe bytecode peephole.

## Close Notes
Strengthened consuming projection selection so prior `load_tag` and `load_object_witness` reads no longer block destruction after their independent scalar results are materialized, while any prior dependent borrow still prevents consumption. This removes the actual borrow-retain-release source pattern before ARC insertion and preserves canonical ownership transfer through block edges; no redundant move instructions remain in the Scheduler output. A finalization snapshot locks the direct `consume_fields` transfer with no inserted retain or release. The Coroutine Scheduler falls from 204 retains plus 169 releases to 18 retains plus 27 releases, an 87.9% reduction, without crossing calls or joins in a post-bytecode peephole. Verification: VM CFG 19/19, LoisVM lowering 35/35, interpreter 21/21, Wasm 45/45, Lane 56/56, all runnable examples, warning-enabled check, and `git diff --check`.
