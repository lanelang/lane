## Description
Add an end-to-end conformance suite proving that effectful Lane programs compile into effect-erased bytecode and execute equivalently in the LoisVM interpreter and Wasmoon-backed Wasm tier.

## Design
Use the Buslane interpreter as the source-semantics oracle where practical, then execute the same lowered image through both LoisVM backends. Inspect complete observable results or output traces rather than asserting rendered substrings. Include ownership-sensitive cases so continuation closure capture and repeated resume exercise ARC behavior.

## Acceptance Criteria
- [ ] Tests cover direct and nested deep handlers.
- [ ] Tests cover final clauses, handler-arm effects, and nearest-handler precedence.
- [ ] Tests cover single and multi-shot resume with captured reference-counted values.
- [ ] Tests cover generic handlers or continuations carrying layout witnesses.
- [ ] Tests cover operations that escape to the primitive runtime host boundary.
- [ ] Interpreter and Wasm outputs agree, and emitted bytecode contains no effect-specific representation.
- [ ] Full `moon test --target native` and `moon check --target all --warn-list +73` pass.
