## Description
The current formatter is width-sensitive but not fully comment-preserving. Migrate it to consume parsed source and trivia view while keeping the existing pretty-printer architecture.

## Design
Extend the current syntax pretty printer with a formatting context that can consume attached trivia. Do not introduce a second formatter and do not merge comments into rendered strings after the fact. Keep v1 full-file and parse-success-only.

## Acceptance Criteria
- [x] `lane fmt` preserves comments and meaningful blank lines for parse-successful source files.
- [x] Existing width-sensitive formatting behavior remains stable.
- [x] Formatter does not use post-render comment reinsertion.
- [x] Range formatting and error-tolerant formatting remain explicitly out of scope.
