## Description
The artifact text reader is already method-based, but the writer side still passes `Array[String]` through many `write_*` free functions. This makes serialization code noisy and asymmetric.

## Design
Introduce a private `ArtifactTextWriter` with methods for primitive writes and artifact sections. Keep the public text format unchanged and avoid compatibility aliases. The public `*_artifact_to_text` functions should construct a writer, call methods, and return `finish()`.

## Acceptance Criteria
- [x] Artifact text writing no longer threads `lines : Array[String]` through writer helper functions.
- [x] `.lmi`, `.lmo`, and `.lbp` text output remains semantically unchanged.
- [x] Existing artifact parser/writer tests pass.
