# ISS-221: Specialize and eliminate static operation dictionaries

## Description
Specialize reachable ordinary type-generic functions and static operation dictionaries so primitive operations do not survive as runtime constructor, projection, closure, and indirect-call chains.

## Design
Use concrete type applications and occurrence facts to clone profitable reachable definitions, then simplify constructor construction and projection through the core optimizer. Inline only when size, recursion, escape, and effect constraints make the rewrite safe. Preserve polymorphic definitions when they remain first-class or have multiple unspecialized uses.

The first vertical slice should reduce the selected Int `Equal` dictionary path to the `%i64_equal` intrinsic without symbol-specific compiler logic.

## Acceptance Criteria
- [x] Concrete reachable dictionary uses specialize through ordinary generic rules.
- [x] Static constructor/projection chains collapse without changing nominal semantics.
- [x] Primitive Int equality reaches the intrinsic without dictionary allocation or indirect dispatch.
- [x] Recursive or escaping polymorphic definitions remain correct.
- [x] Focused generic, dictionary, effect, and executable regressions pass.
