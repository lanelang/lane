## Description
Replace private invokes and installs with statically selected operation callables and ordinary lexical dictionary construction.

## Design
For an install with result `H` and residual row `F`, use the Church-encoded residual computation `M_F<H>` as the handled body's local CPS answer. Project each known operation field from the nearest concrete dictionary and call it with payloads plus the current continuation returning that local answer. Construct handler dictionaries from clause closures, transform the handled body under the inner dictionary binding, and use relay dictionaries to suspend residual operations into `M_F<H>`. Transform operation and final clauses into the residual computation, then run it with the actual outer contexts and continuation. Resume closures re-run the captured inner continuation into a fresh residual computation on every call.

## Acceptance Criteria
- [ ] Invokes select a statically known dictionary field without a runtime operation tag.
- [ ] Installs become ordinary immutable dictionary construction and disappear.
- [ ] Nested same-effect handlers select the nearest lexical dictionary.
- [ ] Clause and final bodies use the outer context while resumes retain the inner context.
- [ ] Handler-arm effects and repeated resume preserve deep multi-shot semantics.
- [ ] Generic operation payloads and hidden parameters retain their transformed types.
