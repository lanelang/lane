## Description
Compiler diagnostics, CLI pretty printing, and LSP diagnostic transport have different presentation requirements. The compiler should own structured diagnostic facts, while CLI and LSP adapters should format those facts for their targets without reinterpreting compiler semantics. Recent LSP work exposed that message-only diagnostics can be too terse for clients that do not display related information.

## Design
Deepen diagnostic presentation as a module that separates structured compiler diagnostics from target-specific presentation. Preserve the rule that diagnostics are published to the real source URI that owns each diagnostic, and avoid leaking LSP transport concerns into `lanec`.

## Acceptance Criteria
- [x] CLI and LSP diagnostic presentation rules are documented or concentrated behind a clear module interface.
- [x] LSP diagnostics preserve real source URI ownership and related information where available.
- [x] CLI diagnostic rendering remains stable unless intentionally changed.
